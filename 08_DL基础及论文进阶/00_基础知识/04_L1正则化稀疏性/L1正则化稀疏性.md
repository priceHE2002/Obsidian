# L1 正则化为何更容易产生稀疏解

> L1 正则化（Lasso）相比 L2 正则化（Ridge）更容易使权重参数恰好为零，从而产生稀疏解，实现自动特征选择。这种差异可以从两个核心角度理解：**几何约束形状**和**梯度特性**。理解这一点，就理解了为什么 L1 在特征选择和高维稀疏建模中不可替代。

## 一、概览

![[L1正则化_01_标题与概览.jpg]]

L1 正则化和 L2 正则化都是防止过拟合的常用技术，但它们的数学形式决定了完全不同的行为：

| | L1 正则化（Lasso） | L2 正则化（Ridge） |
|---|---|---|
| **惩罚项** | $\lambda \sum |w_i|$ | $\lambda \sum w_i^2$ |
| **几何形状** | 菱形（尖角在坐标轴上） | 圆形（光滑边界） |
| **梯度特性** | 恒定力推向零 | 与 |w| 成正比的力 |
| **稀疏性** | ✅ 产生精确零值 | ❌ 仅缩小但不为零 |
| **特征选择** | 自动进行 | 不自动 |

### 直觉类比

想象你在一个房间里寻找最优位置。L1 的约束区域像一个菱形房间——你最容易停在尖角上，而尖角恰好就在坐标轴上（某个维度恰好为零）。L2 的约束区域像一个圆形房间——你几乎不可能恰好停在坐标轴上。

## 二、L1 与 L2 的直观对比

![[L1正则化_02_L2与L1对比.jpg]]

### 2.1 损失函数形式

带正则化的损失函数：

$$L_{\text{L1}}(w) = \underbrace{\frac{1}{n}\sum_{i=1}^{n}(wx_i + b - y_i)^2}_{\text{MSE 损失}} + \underbrace{\lambda \sum_{j} |w_j|}_{\text{L1 惩罚}}$$

$$L_{\text{L2}}(w) = \underbrace{\frac{1}{n}\sum_{i=1}^{n}(wx_i + b - y_i)^2}_{\text{MSE 损失}} + \underbrace{\lambda \sum_{j} w_j^2}_{\text{L2 惩罚}}$$

### 2.2 梯度更新对比

**L2 的梯度更新：**
$$w \leftarrow w - \eta \cdot \frac{\partial \text{MSE}}{\partial w} - \eta \cdot \underbrace{2\lambda w}_{\text{L2梯度}} = (1 - 2\eta\lambda)w - \eta \cdot \frac{\partial \text{MSE}}{\partial w}$$

L2 相当于每次更新时把 $w$ 乘以一个小于 1 的因子 $(1 - 2\eta\lambda)$——这叫**权重衰减**。它让所有权重逐渐缩小，但不会精确变为零。

**L1 的梯度更新：**
$$w \leftarrow w - \eta \cdot \frac{\partial \text{MSE}}{\partial w} - \eta \cdot \underbrace{\lambda \cdot \text{sign}(w)}_{\text{L1梯度}}$$

L1 每次减去一个**恒定量** $\eta\lambda$，与 $w$ 的大小无关。即使 $w$ 已经很小，推力不减，最终推过零点。

```python
import torch
import torch.nn as nn
import matplotlib.pyplot as plt
import numpy as np

# 对比 L1 和 L2 正则化对权重的影响
torch.manual_seed(42)

# 模拟一个简单场景：真实权重稀疏，只有 2 个有效特征
n_features = 100
true_w = torch.zeros(n_features)
true_w[0] = 3.0   # 两个有效特征
true_w[1] = -2.0

# 生成数据
X = torch.randn(200, n_features)
y = X @ true_w + 0.1 * torch.randn(200)

# L1 正则化
model_l1 = nn.Linear(n_features, 1, bias=False)
opt_l1 = torch.optim.SGD(model_l1.parameters(), lr=0.1, weight_decay=0)  # weight_decay 就是 L2!
# 手动实现 L1
def train_with_l1(model, X, y, lr=0.01, l1_lambda=0.05, epochs=500):
    w = model.weight.data.clone()
    for _ in range(epochs):
        pred = X @ w.T
        loss = ((pred - y.unsqueeze(1)) ** 2).mean()
        grad = (2 / len(X)) * X.T @ (pred - y.unsqueeze(1))
        # L1 梯度：sign(w) * lambda
        w = w - lr * grad.T - lr * l1_lambda * torch.sign(w)
    model.weight.data = w

train_with_l1(model_l1, X, y, l1_lambda=0.05)
w_l1 = model_l1.weight.data.squeeze()

# L2 正则化（直接用 weight_decay）
model_l2 = nn.Linear(n_features, 1, bias=False)
opt_l2 = torch.optim.SGD(model_l2.parameters(), lr=0.1, weight_decay=0.1)
for _ in range(500):
    pred = model_l2(X)
    loss = ((pred - y.unsqueeze(1)) ** 2).mean()
    opt_l2.zero_grad()
    loss.backward()
    opt_l2.step()
w_l2 = model_l2.weight.data.squeeze()

print(f"L1: 零权重个数 = {(w_l1.abs() < 1e-3).sum().item()} / {n_features}")
print(f"L1: 最大权重 = {w_l1.abs().max():.4f}")
print(f"L2: 零权重个数 = {(w_l2.abs() < 1e-3).sum().item()} / {n_features}")
print(f"L2: 最大权重 = {w_l2.abs().max():.4f}")
```

## 三、几何角度解释

![[L1正则化_03_几何角度解释.jpg]]

### 3.1 约束区域形状

将正则化问题转化为约束优化问题（以二维为例）：

**L2 约束区域：** $w_1^2 + w_2^2 \leq t$ → 圆形，边界处处光滑

**L1 约束区域：** $|w_1| + |w_2| \leq t$ → 菱形，在坐标轴上有尖角

关键：最优解是 MSE 损失等高线与约束区域的**首次交点**。

- 圆形区域：等高线可以在圆周上任意点首次相切，几乎不会恰好切在坐标轴上
- 菱形区域：等高线极大概率首先碰到**尖角**，而尖角恰好就在坐标轴上（某维度 $w_j = 0$）

### 3.2 为什么尖角导致稀疏？

![[L1正则化_04_几何直观.jpg]]

想象 MSE 等高线是一族椭圆。当椭圆逐渐扩大（对应损失增大），第一次碰到约束边界的位置：

- 对于圆形（L2）：在圆周上均匀分布，大多数交点两个权重都不为零
- 对于菱形（L1）：菱形的四个顶点在坐标轴上，椭圆最容易碰到顶点位置 → $w_1=0$ 或 $w_2=0$

这就是为什么 L1 产生稀疏解的**几何本质**：菱形的尖角使最优解天然落在坐标轴上。

## 四、梯度角度解释

![[L1正则化_05_梯度角度解释.jpg]]

### 4.1 L1 梯度是恒定的

从子梯度（subgradient）的角度看：

$$\frac{\partial}{\partial w_j} \lambda |w_j| = \begin{cases} \lambda, & w_j > 0 \\ [-\lambda, \lambda], & w_j = 0 \\ -\lambda, & w_j < 0 \end{cases}$$

无论 $w_j$ 多大或多小，L1 惩罚的梯度恒定不变。这意味着：
- 即使 $w_j$ 已经非常接近零，推力仍是 $\lambda$
- 当 $|x(wx + b - y)| < \lambda$ 时，MSE 梯度不足以抵消 L1 的推力，$w$ 就会被直接推过零点

### 4.2 L2 梯度随 |w| 衰减

$$\frac{\partial}{\partial w_j} \lambda w_j^2 = 2\lambda w_j$$

当 $w_j$ 趋近于零时，梯度也趋近于零，推动力越来越小，形成渐进趋近行为——永远到达不了零。

![[L1正则化_06_梯度对比.jpg]]

**梯度特性的核心差异：**

| 特性 | L1（Lasso） | L2（Ridge） |
|------|------------|------------|
| 梯度大小 | 恒定 $\lambda$ | $2\lambda w$，随 $|w|$ 缩小 |
| 对小权重的推力 | 强（恒定不变） | 弱（正比缩小） |
| 能否推过零点 | ✅ 可以 | ❌ 不能（渐近行为） |
| 达到零的条件 | $|x(wx+b-y)| < \lambda$ | 永远不能精确为零 |

## 五、两者各自适用场景

L1 和 L2 并非一个"优于"另一个，而是适用不同场景：

**L1 正则化更适用：**
- 高维数据（特征数远大于样本数），需要自动特征选择
- 已知只有少数特征真正起作用（稀疏先验）
- 需要可解释的模型——知道哪些特征被选中
- 例如：基因表达数据分析（数万基因中只有几十个与疾病相关）

**L2 正则化更适用：**
- 所有特征都有一定贡献，不希望任何特征被完全排除
- 多重共线性严重的数据（L2 可以稳定矩阵求逆）
- 深度学习中的 weight decay 防止权重过大
- 例如：图像分类中每个像素都对预测有微小贡献

**Elastic Net（L1 + L2 组合）：**
$$L = \text{MSE} + \lambda_1 \sum |w_j| + \lambda_2 \sum w_j^2$$
兼顾稀疏性和稳定性，特别适合特征之间高度相关的情况。

## 六、总结

![[L1正则化_07_总结.jpg]]

L1 比 L2 更容易产生稀疏解，根本原因有两个：

1. **几何角度**：L1 的约束区域是菱形，尖角在坐标轴上。损失等高线与约束区域的首次交点大概率在尖角上，自然产生零值。

2. **梯度角度**：L1 梯度的恒定推力不会因权重变小而减弱，能够在 MSE 梯度较小时将权重直接推过零点。L2 梯度随权重缩小而同步减弱，只能渐近趋零。

这两个角度是同一枚硬币的两面——它们共同解释了 L1 正则化为什么是"稀疏性之王"。
