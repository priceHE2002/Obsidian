# 端到端流程

本文档追踪一个完整的请求从用户调用 `llm.generate()` 到返回结果的每一步，串联所有模块。

## 示例代码

```python
from nanovllm import LLM, SamplingParams

llm = LLM("/path/to/Qwen3-0.6B", enforce_eager=True, tensor_parallel_size=1)
sampling_params = SamplingParams(temperature=0.6, max_tokens=256)
prompts = ["Hello, Nano-vLLM."]
outputs = llm.generate(prompts, sampling_params)
print(outputs[0]["text"])
```

## 阶段 0：初始化

```
LLM(path, enforce_eager=True, tensor_parallel_size=1)
  └─ LLMEngine.__init__(model=path, ...)
       ├─ Config(model=path) → 加载 hf_config, 验证参数
       ├─ Sequence.block_size = 256
       ├─ [TP>1] 启动 Worker 进程
       ├─ ModelRunner(config, rank=0, events)
       │    ├─ dist.init_process_group("nccl", ...)
       │    ├─ torch.set_default_device("cuda")
       │    ├─ Qwen3ForCausalLM(hf_config)  ← 创建模型结构
       │    ├─ load_model(model, path)      ← 加载 HuggingFace 权重
       │    │    └─ 逐 safetensors 文件加载, weight_loader 处理 TP 分片
       │    ├─ warmup_model()              ← 一次假 Prefill, 触发 JIT 编译
       │    ├─ allocate_kv_cache()         ← 计算可用显存, 分配 KV-Cache
       │    │    └─ kv_cache: [2, L, B, 256, H, D]
       │    └─ capture_cudagraph()         ← [enforce_eager=False]
       │         └─ 捕获 33 个 CUDA Graph (batch 1..512)
       ├─ AutoTokenizer.from_pretrained()   ← 加载 tokenizer
       └─ Scheduler(config)
            └─ BlockManager(num_blocks, 256)
       └─ atexit.register(exit)
```

## 阶段 1：添加请求

```python
llm.generate(["Hello, Nano-vLLM."], SamplingParams(...))
  ├─ for prompt, sp in zip(prompts, sampling_params):
  │    add_request(prompt, sp)
  │    ├─ tokenizer.encode(prompt)
  │    │    → [151645, 9707, 11, ...]  (token IDs)
  │    ├─ Sequence(token_ids, sampling_params)
  │    │    ├─ seq_id = 0, status = WAITING
  │    │    ├─ num_tokens = num_prompt_tokens
  │    │    ├─ num_cached_tokens = 0
  │    │    └─ block_table = []
  │    └─ scheduler.add(seq)
  │         └─ waiting.append(seq)
  └─ while not is_finished():  ← 进入主循环
```

## 阶段 2：第一轮 Step — Prefill

### 2a. 调度

```
step()
  └─ scheduler.schedule()
       ├─ 从 waiting 取 seq_0
       ├─ block_manager.can_allocate(seq_0)
       │    ├─ 遍历 Block 0...(n-2), 计算链式哈希
       │    ├─ hash_to_block_id 查找（首次为空，num_cached_blocks=0）
       │    └─ free_block_ids >= num_blocks → OK
       ├─ block_manager.allocate(seq_0, num_cached_blocks=0)
       │    └─ 分配 num_blocks 个新 Block（如 seq 有 8 tokens: 1 block）
       │         seq.block_table = [0]
       ├─ seq.num_scheduled_tokens = 8
       ├─ Prefill 完成 → seq.status = RUNNING, waiting→running
       └─ return [seq_0], True  (is_prefill=True)
```

### 2b. GPU 推理 — Prefill 准备

```
model_runner.call("run", [seq_0], True)
  └─ run(seqs, is_prefill=True)
       └─ prepare_prefill([seq_0])
            ├─ input_ids = seq[0:8]  (全部 8 个 token)
            ├─ positions = [0,1,2,3,4,5,6,7]
            ├─ cu_seqlens_q = [0, 8]
            ├─ cu_seqlens_k = [0, 8]
            ├─ slot_mapping =
            │    token 0..7 → Block 0 × 256 + 0..7 → [0,1,2,3,4,5,6,7]
            └─ set_context(True, cu_seqlens_q, cu_seqlens_k,
                           max_seqlen_q=8, max_seqlen_k=8,
                           slot_mapping, ..., block_tables=None)
```

### 2c. GPU 推理 — 模型前向

```
run_model(input_ids=[8], positions=[0..7], is_prefill=True)
  └─ model(input_ids, positions)  (走标准前向)
       ├─ embed_tokens(input_ids) → [8, hidden_dim]
       ├─ for each DecoderLayer:
       │    ├─ RMSNorm(x, residual) → [8, hidden], residual
       │    ├─ Qwen3Attention:
       │    │    ├─ qkv_proj(x) → q,k,v [8, heads, head_dim]
       │    │    ├─ q_norm, k_norm (Qwen3 特有)
       │    │    ├─ RoPE(positions, q, k)
       │    │    ├─ Attention(q, k, v):
       │    │    │    ├─ store_kvcache(k, v, slot_mapping)  ← Triton kernel
       │    │    │    └─ flash_attn_varlen_func(q, k, v, ...)
       │    │    └─ o_proj(o) → [8, hidden]
       │    ├─ RMSNorm(hidden, residual)
       │    └─ MLP: gate_up_proj → SiluAndMul → down_proj
       └─ final RMSNorm → hidden_states [8, hidden]
  └─ compute_logits(hidden_states)
       └─ lm_head: 只取最后一个 token → logits [1, vocab_size]
```

### 2d. 采样

```
sampler(logits [1, vocab_size], temperatures [0.6])
  ├─ logits / 0.6 → softmax → probs
  ├─ Gumbel-Max: probs / Exponential(1) → argmax
  └─ token_ids = [12345]  (sampled token)
```

### 2e. 后处理

```
scheduler.postprocess([seq_0], [12345], is_prefill=True)
  ├─ block_manager.hash_blocks(seq_0)
  │    └─ Block 0 写满 → compute_hash([151645,9707,...], -1) → 写入 hash_to_block_id
  ├─ seq.num_cached_tokens += 8 → 8
  ├─ is_prefill 且 num_cached_tokens(8) == num_tokens(8) ✓
  ├─ seq.append_token(12345)
  │    └─ token_ids = [151645,..., 12345], num_tokens = 9
  └─ 未达 max_tokens(256), 非 EOS → 继续 RUNNING
```

## 阶段 3：后续轮次 Step — Decode

### 3a. 调度

```
scheduler.schedule()
  ├─ waiting 为空 → 进入 Decode
  ├─ 从 running 取 seq_0
  ├─ block_manager.can_append(seq_0)
  │    └─ len(seq_0)=9, 9%256=9≠1 → 不需要新 Block → OK
  ├─ seq.num_scheduled_tokens = 1
  └─ return [seq_0], False  (is_prefill=False)
```

### 3b. GPU 推理 — Decode 准备

```
prepare_decode([seq_0])
  ├─ input_ids = [12345]  (最后一个 token)
  ├─ positions = [8]      (位置 8)
  ├─ slot_mapping = [8]   (Block 0 × 256 + 8)
  ├─ context_lens = [9]   (完整序列长度)
  ├─ block_tables = [[0, -1, -1, ...]]  (第一个序列只有 1 个有效 block)
  └─ set_context(False, slot_mapping, context_lens, block_tables)
```

### 3c. GPU 推理 — CUDA Graph (首选路径)

```
run_model(input_ids=[1], positions=[8], is_prefill=False)
  ├─ bs=1, enforce_eager=False, bs≤512
  ├─ graph = self.graphs[1]  (使用 bs=1 的预捕获图)
  ├─ 更新 graph_vars:
  │    input_ids[:1] = [12345]
  │    positions[:1] = [8]
  │    slot_mapping: fill_(-1), [:1] = [8]
  │    context_lens: zero_(), [:1] = [9]
  │    block_tables[:1] = [[0, -1, ...]]
  ├─ graph.replay()  ← 单次 kernel launch 执行整个前向！
  │    └─ 内部: embed → 28×DecoderLayer → final norm
  │         每个 Attention 层:
  │           store_kvcache(只有 1 token)
  │           flash_attn_with_kvcache(q.unsqueeze(1), k_cache, v_cache,
  │                                    cache_seqlens=[9], block_table=[[0]])
  └─ compute_logits(outputs[:1]) → logits [1, vocab]
```

### 3d. 采样与后处理

```
sampler(logits, [0.6]) → token_ids = [6789]
postprocess: append_token(6789) → num_tokens=10 → 继续循环
```

## 阶段 4：生成完成

```
第 256 次 Decode 后:
  seq.num_completion_tokens == 256 → max_tokens 达到
  postprocess:
    ├─ seq.status = FINISHED
    ├─ block_manager.deallocate(seq_0)
    │    └─ Block 0 ref_count: 1→0 → 回收 → free_block_ids
    └─ running.remove(seq_0)

generate() 循环:
  ├─ step() 返回 output = [(0, [12345, 6789, ...])]
  ├─ outputs[0] = [12345, 6789, ...]
  ├─ pbar.update(1)
  └─ is_finished() → True (waiting + running 都为空)

tokenizer.decode([12345, 6789, ...]) → "Hello! I'm Nano-vLLM..."
return [{"text": "Hello! I'm Nano-vLLM...", "token_ids": [...]}]
```

## 完整调用栈总结

```
llm.generate(prompts, sampling_params)
  │
  ├─ add_request(prompt, sp)
  │   ├─ tokenizer.encode(prompt)
  │   ├─ Sequence(token_ids, sp)
  │   └─ scheduler.add(seq)
  │
  └─ while not is_finished():
       └─ step()
            ├─ scheduler.schedule()
            │   ├─ [Prefill]  can_allocate → allocate → num_scheduled_tokens
            │   └─ [Decode]   can_append → may_append → num_scheduled_tokens=1
            │
            ├─ model_runner.call("run", seqs, is_prefill)
            │   └─ run(seqs, is_prefill)
            │       ├─ prepare_prefill/decode()
            │       │   └─ set_context(...)
            │       ├─ [TP] write_shm("run", seqs, is_prefill)
            │       │       → Worker 进程 read_shm → run()
            │       ├─ run_model(input_ids, positions, is_prefill)
            │       │   ├─ [Prefill] model(input_ids, positions)
            │       │   └─ [Decode]  graph.replay() 或 model.forward()
            │       ├─ compute_logits(hidden_states)
            │       └─ sampler(logits, temperatures)
            │           └─ Gumbel-Max 采样
            │
            └─ scheduler.postprocess(seqs, token_ids, is_prefill)
                ├─ hash_blocks(seq)
                ├─ append_token(token_id)
                ├─ [完成] deallocate(seq)
                └─ return completed sequences
```
