# 采样器

`sampler.py` 实现了温度调节的随机采样。代码仅 12 行，但实现了一个关键的采样技巧。

## Sampler 实现

```python
class Sampler(nn.Module):
    @torch.compile
    def forward(self, logits, temperatures):
        logits = logits.float().div_(temperatures.unsqueeze(dim=1))
        probs = torch.softmax(logits, dim=-1)
        sample_tokens = probs.div_(
            torch.empty_like(probs).exponential_(1).clamp_min_(1e-10)
        ).argmax(dim=-1)
        return sample_tokens
```

## 温度调节

```python
logits = logits.float().div_(temperatures.unsqueeze(dim=1))
```

温度 T 控制采样的随机性：
- T < 1.0：概率分布更尖锐，输出更确定性（接近 greedy）
- T = 1.0：标准 softmax 采样
- T > 1.0：概率分布更平坦，输出更随机

注意项目限制 `temperature > 1e-10`（不支持 greedy），因为 `div_` 除以接近 0 的值会导致数值溢出。

## Gumbel-Max Trick

```python
probs.div_(torch.empty_like(probs).exponential_(1).clamp_min_(1e-10)).argmax(dim=-1)
```

这是 `sample_tokens` 实现的核心：**Gumbel-Max Trick**。它不是从 categorical 分布中采样的唯一方法，但在推理引擎中很流行，因为它避免了显式计算 CDF 或使用 `torch.multinomial`。

### 原理

Gumbel-Max Trick：从分布 `p` 中采样等价于：

```
argmax_i { log(p_i) + G_i }
其中 G_i ~ Gumbel(0, 1)，即 G_i = -log(-log(U_i)), U_i ~ Uniform(0,1)
```

等价于：

```
argmax_i { logits_i / T + G_i } = argmax_i { probs_i / exp(-G_i) }
其中 exp(-G_i) ~ Exponential(1)
```

nano-vllm 的实现就是应用了这个等价变换：

```python
# Gumbel 噪声: torch.empty_like(probs).exponential_(1)
# argmax_i { probs_i / E_i } 等价于从 softmax(logits/T) 中随机采样
```

`clamp_min_(1e-10)` 防止除以 0（理论上 Exponential(1) 可以产生任意接近 0 的值，概率虽小但需防范）。

## 为什么不直接用 torch.multinomial？

`torch.multinomial` 对大批量采样性能不佳，因为它需要为每个样本计算累积分布函数。Gumbel-Max Trick 将所有操作转化为向量化的逐元素运算 + argmax，CUDA kernel 效率更高。`@torch.compile` 进一步将 `div_`、`softmax`、`div_`、`exponential_`、`argmax` 融合优化。
