# 线性层与张量并行

`linear.py` 实现了 5 种线性层变体，覆盖了张量并行（Tensor Parallelism）的所有必要模式。这是 nano-vllm 最精巧的模块化设计之一。

## 线性层体系

```
LinearBase (抽象基类)
├── ReplicatedLinear        ← 普通 Linear，不切分
├── ColumnParallelLinear    ← 按输出列切分
│   ├── MergedColumnParallelLinear  ← 合并多个 ColumnParallel (gate+up)
│   └── QKVParallelLinear           ← QKV 合并投影
└── RowParallelLinear       ← 按输入列切分，输出需 all_reduce
```

## LinearBase — 抽象基类

```python
class LinearBase(nn.Module):
    def __init__(self, input_size, output_size, bias=False, tp_dim=None):
        self.tp_dim = tp_dim     # 切分的维度：0=输出, 1=输入, None=不切
        self.tp_rank = dist.get_rank()
        self.tp_size = dist.get_world_size()
        self.weight = nn.Parameter(torch.empty(output_size, input_size))
        self.weight.weight_loader = self.weight_loader  # 自定义加载器
        if bias:
            self.bias = nn.Parameter(torch.empty(output_size))
            self.bias.weight_loader = self.weight_loader
```

每个权重参数被绑定了一个 `weight_loader` 方法——这是一个巧妙的设计。在 `load_model()` 中：
```python
weight_loader = getattr(param, "weight_loader", default_weight_loader)
weight_loader(param, f.get_tensor(weight_name))
```

不同类型的线性层有不同的加载逻辑（ColumnParallel 需要 slice 权重，QKVParallel 需要按 shard_id 分配），通过将 `weight_loader` 绑定到参数上，loader 不需要知道参数类型，只需调用参数的 `weight_loader` 方法。

## ReplicatedLinear — 不切分的全连接

```python
class ReplicatedLinear(LinearBase):
    def weight_loader(self, param, loaded_weight):
        param.data.copy_(loaded_weight)       # 完整拷贝，每个 rank 持有完整权重

    def forward(self, x):
        return F.linear(x, self.weight, self.bias)
```

每个 GPU 持有完整权重副本。用于不需要并行的层（如 bias 很小的情况），或作为 fallback。

## ColumnParallelLinear — 按输出列切分

```python
class ColumnParallelLinear(LinearBase):
    def __init__(self, input_size, output_size, bias=False):
        tp_size = dist.get_world_size()
        super().__init__(input_size, divide(output_size, tp_size), bias, 0)  # tp_dim=0

    def weight_loader(self, param, loaded_weight):
        shard_size = param_data.size(self.tp_dim)
        start_idx = self.tp_rank * shard_size
        loaded_weight = loaded_weight.narrow(self.tp_dim, start_idx, shard_size)
        param_data.copy_(loaded_weight)

    def forward(self, x):
        return F.linear(x, self.weight, self.bias)
```

**权重切分示意**（2 GPU, output_size=1024）：

```
完整权重: [1024, input_size]
  ├─ GPU 0: [ 512, input_size]  ← 前 512 行
  └─ GPU 1: [ 512, input_size]  ← 后 512 行

前向: y_0 = x @ W_0^T  (每个 GPU 独立计算各自部分)
      输出拼接由后续 RowParallelLinear 的 all_reduce 隐式完成
```

关键：**ColumnParallel 不主动做 all-gather**，因为下一个操作通常是 RowParallelLinear（其 all_reduce 隐式完成了合并）。

## MergedColumnParallelLinear — 合并多路投影

```python
class MergedColumnParallelLinear(ColumnParallelLinear):
    def __init__(self, input_size, output_sizes: list[int], bias=False):
        self.output_sizes = output_sizes     # 如 [intermediate, intermediate]
        super().__init__(input_size, sum(output_sizes), bias)

    def weight_loader(self, param, loaded_weight, loaded_shard_id: int):
        shard_offset = sum(self.output_sizes[:loaded_shard_id]) // self.tp_size
        shard_size = self.output_sizes[loaded_shard_id] // self.tp_size
        param_data = param_data.narrow(self.tp_dim, shard_offset, shard_size)
        loaded_weight = loaded_weight.chunk(self.tp_size, self.tp_dim)[self.tp_rank]
        param_data.copy_(loaded_weight)
```

用于 SwiGLU 的 gate_proj + up_proj 合并。将两个独立的矩阵乘法融合为一个，减少 kernel launch 次数。

```
MF = MergedColumnParallelLinear(hidden, [intermediate, intermediate])

HF 权重:
  gate_proj.weight [intermediate, hidden]  → shard_id=0
  up_proj.weight   [intermediate, hidden]  → shard_id=1

nano-vllm 权重:
  gate_up_proj.weight [2*intermediate//tp, hidden]
  布局: [gate_shard | up_shard]
```

## QKVParallelLinear — QKV 合并投影

```python
class QKVParallelLinear(ColumnParallelLinear):
    def __init__(self, hidden_size, head_size, total_num_heads, total_num_kv_heads):
        self.num_heads = divide(total_num_heads, tp_size)
        self.num_kv_heads = divide(total_num_kv_heads, tp_size)
        output_size = (total_num_heads + 2 * total_num_kv_heads) * head_size
        super().__init__(hidden_size, output_size)

    def weight_loader(self, param, loaded_weight, loaded_shard_id: str):
        assert loaded_shard_id in ["q", "k", "v"]
        if loaded_shard_id == "q":
            shard_offset = 0
            shard_size = self.num_heads * self.head_size
        elif loaded_shard_id == "k":
            shard_offset = self.num_heads * self.head_size
            shard_size = self.num_kv_heads * self.head_size
        else:  # v
            shard_offset = self.num_heads * self.head_size + self.num_kv_heads * self.head_size
            shard_size = self.num_kv_heads * self.head_size
        param_data = param_data.narrow(self.tp_dim, shard_offset, shard_size)
        loaded_weight = loaded_weight.chunk(self.tp_size, self.tp_dim)[self.tp_rank]
        param_data.copy_(loaded_weight)
```

QKV 合并投影是最重要的张量并行优化之一。在 GQA（Grouped Query Attention）中，KV heads 比 Q heads 少，所以需要精确计算每个 shard 的偏移。

```
Qwen3-0.6B example (TP=2, num_heads=16, num_kv_heads=8, head_dim=64):
完整权重: [16*64 + 2*8*64, hidden] = [2048, hidden]
  Q 部分: [1024, hidden]  → 每个 GPU: [512, hidden]
  K 部分: [512, hidden]   → 每个 GPU: [256, hidden]
  V 部分: [512, hidden]   → 每个 GPU: [256, hidden]

GPU 0 的 qkv_proj.weight: [1024, hidden]
  布局: [Q0...Q511 | K0...K255 | V0...V255]
```

## RowParallelLinear — 按输入列切分 + all_reduce

```python
class RowParallelLinear(LinearBase):
    def __init__(self, input_size, output_size, bias=False):
        super().__init__(divide(input_size, tp_size), output_size, bias, 1)  # tp_dim=1

    def weight_loader(self, param, loaded_weight):
        shard_size = param_data.size(self.tp_dim)
        start_idx = self.tp_rank * shard_size
        loaded_weight = loaded_weight.narrow(self.tp_dim, start_idx, shard_size)
        param_data.copy_(loaded_weight)

    def forward(self, x):
        y = F.linear(x, self.weight, self.bias if self.tp_rank == 0 else None)
        if self.tp_size > 1:
            dist.all_reduce(y)    # 跨 GPU 求和
        return y
```

**权重切分示意**（2 GPU, input_size=1024, output_size=512）：

```
完整权重: [512, 1024]
  ├─ GPU 0: [512,  512]  ← 前 512 列
  └─ GPU 1: [512,  512]  ← 后 512 列

前向:
  y_0 = x_0 @ W_0^T  (x_0 是 ColumnParallel 的输出前 512 维)
  y_1 = x_1 @ W_1^T  (x_1 是 ColumnParallel 的输出后 512 维)
  y = all_reduce(y_0 + y_1)   ← 求和得到完整输出
```

RowParallelLinear 的 `all_reduce` 与 ColumnParallelLinear 的分片输出是完美配对的：ColumnParallel 产生按列分片的中间激活，RowParallel 的 `all_reduce` 将它们合并回完整结果。

## 完整的 TP 前向流程

以 Attention 层为例：

```
x [batch, hidden]  ← 每个 GPU 有完整的输入（来自 all_reduce 或 broadcast）
    │
    ▼
QKVParallelLinear  ← ColumnParallel: 每个 GPU 计算自己份额的 QKV
    │
    ▼
q, k, v [batch, heads/tp, head_dim]  ← 各 GPU 只有部分 head
    │
    ▼
Attention (FlashAttention)  ← 每个 GPU 独立计算注意力
    │
    ▼
o [batch, heads/tp * head_dim]  ← 部分 head 的输出
    │
    ▼
RowParallelLinear  ← 每个 GPU 乘自己份额的权重 + all_reduce
    │
    ▼
output [batch, hidden]  ← 完整输出，所有 GPU 一致
```
