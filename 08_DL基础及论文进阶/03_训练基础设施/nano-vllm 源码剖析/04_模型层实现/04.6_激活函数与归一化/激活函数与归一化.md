# 激活函数与归一化

nano-vllm 实现了两个底层组件：`SiluAndMul`（SwiGLU 的门控激活）和 `RMSNorm`（带 fused residual add 的层归一化）。

## SiluAndMul — SwiGLU 激活

```python
class SiluAndMul(nn.Module):
    @torch.compile
    def forward(self, x):
        x, y = x.chunk(2, -1)    # 沿最后一维均分为 gate 和 up
        return F.silu(x) * y     # SiLU(gate) * up
```

SwiGLU 的标准公式是 `SiLU(xW_gate) ⊙ (xW_up)`，其中 `W_gate` 和 `W_up` 被 `MergedColumnParallelLinear` 合并为一个矩阵。所以输入 `x` 的最后一维 = `2 * intermediate_size // tp_size`。

`@torch.compile` 在这里将 `chunk` + `silu` + `mul` 三个操作融合为单个 CUDA kernel，避免多次显存读写。

## RMSNorm — 带 fused residual add

```python
class RMSNorm(nn.Module):
    def __init__(self, hidden_size, eps=1e-6):
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(hidden_size))

    @torch.compile
    def rms_forward(self, x):
        orig_dtype = x.dtype
        x = x.float()
        var = x.pow(2).mean(dim=-1, keepdim=True)
        x.mul_(torch.rsqrt(var + self.eps))   # 原地操作，节省内存
        x = x.to(orig_dtype).mul_(self.weight)
        return x

    @torch.compile
    def add_rms_forward(self, x, residual):
        orig_dtype = x.dtype
        x = x.float().add_(residual.float())   # fused: 先做 residual add
        residual = x.to(orig_dtype)            # 保存新的 residual
        var = x.pow(2).mean(dim=-1, keepdim=True)
        x.mul_(torch.rsqrt(var + self.eps))
        x = x.to(orig_dtype).mul_(self.weight)
        return x, residual
```

### RMSNorm 公式

与 LayerNorm 不同，RMSNorm 不减去均值，只做缩放：

```
RMSNorm(x) = x / sqrt(mean(x²) + ε) * weight
```

对比 LayerNorm `(x - mean(x)) / sqrt(var(x) + ε) * weight + bias`，RMSNorm 少了减均值和加 bias 两步，计算量大幅减少。

### Fused Residual Add 的设计

`add_rms_forward` 将 residual add 和 RMSNorm 融合为一个 kernel：

```
标准实现（3 个 kernel）:
  hidden = hidden + residual          ← kernel 1: elementwise add
  residual = hidden                   ← kernel 2: copy
  hidden = RMSNorm(hidden)            ← kernel 3: normalize

nano-vllm 实现（1 个 kernel）:
  x = (hidden + residual).float()     ← 同一 kernel 内完成
  residual = x.to(orig_dtype)         ← 保存新 residual
  x = RMSNorm(x)                      ← 继续归一化
```

通过 `@torch.compile`，这三个操作被融合为一个 CUDA kernel，消除了 `hidden` 和 `residual` 的两次额外显存写入。

### forward 分发

```python
def forward(self, x, residual=None):
    if residual is None:
        return self.rms_forward(x)       # 普通 RMSNorm
    else:
        return self.add_rms_forward(x, residual)  # 带 residual add
```

统一的 `forward` 通过检查 `residual` 是否为 None 决定走哪个路径。在 Qwen3 DecoderLayer 中，第一个 RMSNorm 首次调用时 `residual=None`（无历史残差），后续所有调用都带 `residual`。

### 计算精度管理

```python
x = x.float()                          # 提升到 float32 做计算
# ... normalize ...
x = x.to(orig_dtype).mul_(self.weight) # 降回原始 dtype（通常是 bfloat16）
```

RMSNorm 的计算精度对数值稳定性影响很大。nano-vllm 在 float32 中计算方差和倒数平方根，避免了 bfloat16 下的累积误差。最后一步 `to(orig_dtype).mul_(weight)` 在低精度下做缩放——因为这是一个逐元素乘法，bfloat16 足够精确。
