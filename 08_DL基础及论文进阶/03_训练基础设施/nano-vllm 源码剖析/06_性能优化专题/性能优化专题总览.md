# 性能优化专题总览

nano-vllm 实现了四大核心性能优化技术，使其在 RTX 4070 Laptop (8GB) 上达到了与 vLLM 相当甚至更优的推理速度（1434 tok/s vs 1362 tok/s）。

| 优化技术 | 加速效果 | 实现位置 | 复杂度 |
|----------|---------|---------|--------|
| CUDA Graph | Decode 延迟降低 30-50% | model_runner.py | 中 |
| Prefix Caching | 共享前缀免 Prefill | block_manager.py | 中 |
| Tensor Parallelism | 多 GPU 线性加速 | model_runner.py + linear.py | 高 |
| Triton Kernel | KV-Cache 写入优化 | attention.py | 低 |
| Torch Compile | 小算子融合加速 | layernorm.py, activation.py, sampler.py | 低 |

## 基准测试结果解读

```
Qwen3-0.6B on RTX 4070 Laptop (8GB)
256 requests, input 100-1024 tokens, output 100-1024 tokens

vLLM:      98.37s, 1361.84 tok/s
nano-vllm: 93.41s, 1434.13 tok/s  (+5.3%)
```

nano-vllm 比 vLLM 快了约 5%，这得益于：
1. **更激进的 `@torch.compile` 使用**：对 RMSNorm、SiLU、Sampler 都用了编译优化
2. **更简洁的代码路径**：少了 vLLM 的 abstraction layers 带来的 overhead
3. **CUDA Graph 的 batch size 分档更细粒度**：`[1,2,4,8, 16,32,48,...,512]` 覆盖所有常见 batch size

但 nano-vllm 也有劣势：不支持 beam search、不支持 multi-modal、只支持 Qwen3 架构、TP 通信使用 pickle 而非 NCCL broadcast（可能在大 TP 时成为瓶颈）。
