# DL 知识库维护 Skill

## 概述

此 Skill 用于维护 `/Users/heyuhang/Documents/Obsidian Vault/08_DL基础及论文进阶/` 目录下的深度学习知识体系。该知识库是 Obsidian Vault 中编号为 `08` 的专题文件夹，以"笔记 + 原文 PDF + 代码实现"三位一体的方式组织，覆盖深度学习基础、具身智能、训练基础设施三大核心领域，共 62 篇论文/主题。

## 适用场景

- 用户需要往知识库中添加一篇新论文（含完整笔记 + PDF + 代码实现）
- 用户需要在 `00_基础知识/` 下新增或扩展某个知识模块（如新增一个语法专题、添加子主题笔记）
- 用户需要在某个领域下创建新的论文专题
- 用户需要更新现有笔记以反映最新理解
- 用户需要为某个专题编写或更新总览文件
- 用户需要检查文件夹结构完整性

## 一、添加新论文——完整工作流程

当用户提供一篇新论文（或指定论文标题/arxiv ID）时，按以下流程操作：

### 步骤 1：确认需求

用 `AskUserQuestion` 确认：
1. 论文标题和 arxiv ID 是什么？
2. 应该放到哪个专题下？（`01_深度学习基础` / `02_具身智能` / `03_训练基础设施`）
3. 用户是否有 PDF（已上传）？如果没有，需要自动下载。
4. 在当前专题下应分配的序号是多少？

### 步骤 2：获取 PDF

**如果用户已提供 PDF**：直接用 `pdf-reading` skill 读取已有 PDF。

**如果用户未提供 PDF**：自动从 arxiv 下载：
1. 优先：`wget -O "{目标路径}/{论文简称} 原文.pdf" "https://arxiv.org/pdf/{arxiv_id}"`
2. 备用：用 `WebSearch` 搜索 `"{论文标题}" pdf` 找下载链接
3. 最终备用：用 `mcp__workspace__web_fetch` 获取 arxiv abs 页面解析关键信息

PDF 保存路径格式：`{用户选定文件夹}/{专题文件夹}/{序号}_{论文简称}/{论文简称} 原文.pdf`

### 步骤 3：阅读 PDF 并提取信息

使用 `pdf-reading` skill 提取：
- 完整论文标题、作者列表、发表年份、会议/期刊
- 一句话核心思想
- 方法/架构创新（详细，含关键公式）
- 关键实验结果（具体数字）
- 局限性
- 与领域内其他论文的关系（用于构建交叉引用）

## 二、添加/扩展基础知识——完整工作流程

当用户需要在 `00_基础知识/` 下新增内容时（如新增一个语法模块、扩展已有模块、添加子主题笔记）：

### 场景 A：新建一个完整模块（如新增 `03_TensorFlow语法/`）

1. **确认需求**：模块名称、主题范围、子主题列表
2. **创建总索引文件** `{序号}_{模块名}/{序号}_{模块名}.md`，包含：
   - 完整 frontmatter（`title`, `tags`, `created`）
   - 目录链接（`[[...]]` 双链）
   - 学习路线建议/整理说明
3. **逐一创建子主题文件夹** `{序号}_{子主题名}/` 及其中的 `.md` 笔记
4. **每个子主题笔记**包含：
   - frontmatter（`title`, `tags`, `created`, `up`）
   - 编号标题（`# 1. 子主题名`）
   - 知识内容 + 可运行代码示例
5. **更新 `00_基础知识/` 层级的全局索引**（如有）

### 场景 B：在已有模块下新增子主题

1. **确定新子主题的位置**（序号应在当前序列末或中间插入）
2. 如果插在中间：调整后续子主题文件夹和文件的序号
3. 创建子主题文件夹和笔记：
   ```bash
   mkdir -p "{路径}/{已有模块}/{序号}_{子主题名}/"
   ```
4. 按子主题笔记模板编写 frontmatter 和正文
5. **更新总索引文件**：在目录列表中添加新 `[[...]]` 链接

### 场景 C：扩展已有子主题（内容补充）

1. 读取已有笔记，理解当前覆盖范围
2. 在适当位置（按章节逻辑）插入新内容
3. 如果需要拆分（内容过长），将现有笔记拆为多个子文件（参考 `06_PyTorch对照/` 的 6.1~6.6 拆分方式）
4. 更新父级索引文件中的目录

### 场景 D：添加对照/对照实现（如 NumPy ↔ PyTorch）

1. 创建对照索引文件（含速查表 `| 操作 | 源框架 | 目标实现 |`）
2. 按功能模块创建子对照文件（编号格式 `{父序号}.{子序号}_{主题}.md`）
3. 在对照索引中声明**加载顺序**（因为子对照文件间有依赖）
4. 每个子对照文件含完整可运行代码（在 Obsidian Execute Code 插件中可直接运行）

## 四、写作风格规范

所有笔记遵循统一的写作风格：

1. **语言**：中文为主，技术术语保留英文（如 "Attention"、"Diffusion"、"Transformer"）
2. **可读性优先**：每个二级标题下第一段给出核心要点概述
3. **具体而非抽象**：使用具体数字（"成功率从 17.3% 提升到 91.7%"），避免"表现很好"
4. **上下文连接**：大量使用 Obsidian 的 `[[...]]` 双链语法引用其他论文
5. **批判性视角**：不仅描述论文做了什么，还要指出不足和为什么被超越
6. **blockquote 高亮**：每篇笔记开头用 `>` 写一句核心定位——读者最应该记住的一句话
7. **表格优先**：对比不同方法/模型时使用表格，让信息一目了然
8. **LaTeX 公式**：数学部分使用 `$...$`（行内）或 `$$...$$`（行间）呈现
9. **禁止内容**：不要逐字翻译论文摘要、不要使用过度复杂的 Mermaid 图（单篇笔记中）、不要在"对你的启示"部分给出与用户背景无关的泛泛之谈

## 五、标签体系

所有笔记使用统一的 Obsidian 标签体系：

### 基础标签
- `论文` — 所有论文详解笔记必须包含
- `代码` — 所有 `model.py.md` 文件必须包含
- `基础知识` — 所有 `00_基础知识/` 下的笔记必须包含

### 领域标签（按专题）
- `深度学习基础` — 01_深度学习基础 总览
- `具身智能` — 02_具身智能 总览
- `训练基础设施` — 03_训练基础设施 总览

### 具体领域/方法标签（示例）
- 架构：`Transformer架构`、`注意力机制`、`自注意力`、`位置编码`、`CNN`
- VLA 相关：`VLA`、`WAM`、`开源`、`里程碑`
- 训练相关：`参数高效微调`、`低秩分解`、`PEFT`、`量化`、`分布式并行`
- 方法相关：`LoRA`、`Flow-Matching`、`Diffusion`、`RL微调`、`Rectified-Flow`
- 其他：`轻量模型`、`PyTorch`、`深度学习`

### 会议/期刊标签
格式：`{会议缩写}{年份}`，如 `NeurIPS2017`、`ICLR2022`、`CVPR2026`、`CoRL2023`、`ICML2023`、`RSS2026`

## 六、完整性检查清单

当被要求"检查文件夹"或"检查完整性"时，对照以下清单：

### 对 `00_基础知识/` 的检查
- [ ] 每个模块有总索引文件（如 `00_PyTorch语法.md`）且 frontmatter 含 `title`、`tags`、`created`
- [ ] 每个子主题文件夹只有一个 `.md` 笔记文件（无 PDF、无 `model.py.md`）
- [ ] 每个子主题笔记有 `up: "[[父索引|显示名]]"` 层级导航链接
- [ ] 总索引中的 `[[...]]` 目录链接全部有效
- [ ] 子主题笔记标题含编号（`# 1. 主题名`），与总索引目录编号一致
- [ ] 对照类模块（如 `06_PyTorch对照/`）的索引文件含速查表和加载顺序说明
- [ ] 面试题类模块（如 `02_深度学习面试题/`）的 `assets/` 中图片用 `![[图片名]]` 正确引用
- [ ] 无孤立图片（在 `assets/` 但未被任何笔记引用）

### 对每个论文子文件夹检查
- [ ] `{论文简称}.md` 存在且包含完整 frontmatter（tags、paper_title、paper_authors、paper_year、paper_venue、paper_url）
- [ ] `{论文简称} 原文.pdf` 存在
- [ ] `model.py.md` 存在且包含 `代码` 和 `PyTorch` 标签
- [ ] `{论文简称}.md` 的六个章节（一~六）完整
- [ ] `{论文简称}.md` 末尾有 `## PDF` 部分和 `[[...]]` PDF 链接

### 对每个专题检查
- [ ] 专题总览文件（`{专题名}论文精讲.md`）存在
- [ ] 总览文件中的论文列表与实际子文件夹一致（数量、序号、名称）
- [ ] 总览文件中的交叉引用链接有效（指向的子文件夹和 `.md` 文件存在）
- [ ] 论文序号连续且按时间线排列

### 全局检查
- [ ] 文件夹之间无重复论文
- [ ] 所有 Obsidian `[[...]]` 链接指向的文件存在
- [ ] 无孤立 PDF（有 PDF 无笔记）或无孤立笔记（有笔记无 PDF）

## 七、与其他 Skill 的协作

- **阅读 PDF**：使用 `pdf-reading` skill 提取论文内容
- **下载论文**：使用 `mcp__workspace__bash` 的 `wget` 从 arxiv 下载
- **创建文件**：使用 `Write` 工具在对应的 Obsidian Vault 路径下创建文件
- **更新总览**：使用 `Edit` 工具在现有总览文件中插入新论文条目

## 八、注意事项

1. **不要修改 `.obsidian/`、`.claude/`、`.omc/` 等隐藏配置目录**
2. **PDF 命名必须包含"原文"二字**：`{论文简称} 原文.pdf`，这是总览文件自动发现 PDF 的约定
3. **代码文件命名固定为 `model.py.md`**，不可用其他名称
4. **所有文件路径必须使用绝对路径**（基于用户的 Obsidian Vault 路径）
5. **Frontmatter 中的 `created` 日期使用实际创建日期**，格式为 `YYYY-MM-DD`
6. **总览文件的 `updated` 字段在每次更新后必须同步更新**
7. **在总览中新增论文时，确保调整所有受影响章节**（时间线、分组、知识图谱、硬件表、贡献表）
