## 1. 指令系统与寻址

### 1.1 指令的基本组成

一条机器指令通常由两类字段组成：

- 操作码 `OP`：说明要执行什么操作，如加、减、访存、转移。
- 地址码 / 操作数字段：说明操作数在哪里、结果写到哪里、下一条指令在哪里。

常见指令格式可以按地址码个数分为零地址、一地址、二地址、三地址、四地址指令。

### 1.2 四地址指令

四地址指令格式：

```text
OP | A1 | A2 | A3 | A4
```

含义：

```text
(A1) OP (A2) -> A3
A4 = 下一条将要执行指令的地址
```

其中：

- `A1`：第一个源操作数地址。
- `A2`：第二个源操作数地址。
- `A3`：结果存放地址。
- `A4`：下一条指令地址。

完成一条四地址指令通常需要访存 4 次：

```text
取指 -> 读 A1 -> 读 A2 -> 写 A3
```

普通顺序执行时，取指后通常有：

```text
(PC) + 1 -> PC
```

四地址指令执行后不是简单顺序执行，而是：

```text
A4 -> PC
```

也就是说，四地址指令把“下一条指令地址”显式写在指令内部。

> 易错点：四地址指令中的 `A4` 是“下址”，不是操作数地址，也不是结果地址。

### 1.3 指令寻址

指令寻址：确定下一条欲执行指令的地址。这个地址始终由程序计数器 `PC` 给出。

顺序寻址时，CPU 按顺序取指：

```text
取指后更新 PC
```

定长指令中常见写法：

```text
(PC) + 1 -> PC
```

变长指令中，CPU 先读入一个字，根据操作码判断该指令总字节数 `n`，再修改 PC：

```text
(PC) + n -> PC
```

示例：若系统采用变长指令字结构，指令字长 = 存储字长 = 16 bit = 2 B，主存按字节编址，则指令地址可能按 0、2、4、6 这样增长。若某条指令占多个字，则 CPU 可能多次访存，每次读入一个字。

> 易错点：变长指令不能机械写成 `PC+1`，应根据本条指令长度更新为 `PC+n`。

### 1.4 数据寻址

数据寻址关注的是操作数在哪里。典型指令结构：

```text
操作码 OP | 寻址特征 | 形式地址 A
```

寻址特征位用于指出寻址方式，形式地址 `A` 需要结合寻址方式解释。

常见数据寻址方式：

| 寻址特征示例 | 寻址方式 | 说明 |
|---|---|---|
| `0000` | 隐含寻址 | 操作数地址隐含在操作码或约定寄存器中 |
| `0001` | 立即寻址 | 形式地址字段本身就是操作数 |
| `0010` | 直接寻址 | 形式地址就是有效地址 |
| `0011` | 间接寻址 | 形式地址指向的主存单元中存放有效地址 |
| `0100` | 寄存器寻址 | 操作数在寄存器中 |
| `0101` | 寄存器间接寻址 | 寄存器中存放有效地址 |
| `0110` | 相对寻址 | 有效地址 = PC + 形式地址 |
| `0111` | 基址寻址 | 有效地址 = 基址寄存器 + 形式地址 |
| `1000` | 变址寻址 | 有效地址 = 变址寄存器 + 形式地址 |
| `1001` | 堆栈寻址 | 操作数在栈中，隐含使用 SP |

### 1.5 变址寻址的作用

变址寻址特别适合数组与循环。

例如：

```c
for (int i = 0; i < 10; i++) {
    sum += a[i];
}
```

可以把数组首地址放在形式地址字段，把下标偏移放在变址寄存器 `IX` 中：

```text
有效地址 = 数组首地址 + (IX)
```

每次循环只需要修改 `IX`，即可访问 `a[0]`、`a[1]`、...、`a[9]`。

示例思想：

```text
ACC <- 0
IX  <- 0
ACC <- ACC + M[数组首地址 + IX]
IX  <- IX + 1
比较 IX 与 10
若未结束则跳回循环
```

### 1.6 堆栈寻址

堆栈寻址：操作数存放在堆栈中，隐含使用堆栈指针 `SP` 作为操作数地址。

堆栈遵循后进先出 `LIFO` 原则。`SP` 指向栈顶元素。

若栈顶在小地址方向，常见操作为：

```text
出栈：
(Msp) -> ACC
(SP) + 1 -> SP

入栈：
(SP) - 1 -> SP
(Y) -> Msp
```

若栈顶在大地址方向，则 `SP` 的加减方向相反。

> 易错点：不同教材可能画成“栈底在上、栈顶在下”，也可能相反。判断入栈/出栈时要看栈顶增长方向，而不是只看图形上下。

### 1.7 扩展操作码

扩展操作码用于在有限指令长度内同时支持不同地址数的指令。

设计扩展操作码时必须注意：

1. 不允许短码是长码的前缀。
2. 各指令的操作码不能重复。

常见设计思想：

- 使用频率高的指令分配较短操作码。
- 使用频率低的指令分配较长操作码。
- 这样可以减少平均译码与分析时间。

例：若基本格式为：

```text
OP | A1 | A2 | A3
```

可以设计为：

```text
0000 ~ 1110        -> 15 条三地址指令
1111 0000 ~ 1111 1110 -> 15 条二地址指令
1111 1111 0000 ~ 1111 1111 1110 -> 15 条一地址指令
1111 1111 1111 0000 ~ 1111 1111 1111 1111 -> 16 条零地址指令
```

### 1.8 常见算术运算指令

Intel 风格中，通常写作：

```asm
op d, s
```

其中：

- `d`：destination，目的操作数。
- `s`：source，源操作数。
- 目的操作数 `d` 不能是常量，因为结果要写回 `d`。

常见指令：

| 功能 | 汇编指令 | 含义 |
|---|---|---|
| 加 | `add d, s` | `d <- d + s` |
| 减 | `sub d, s` | `d <- d - s` |
| 无符号乘 | `mul d, s` | 无符号乘法，乘积存入 `d` |
| 有符号乘 | `imul d, s` | 有符号乘法，乘积存入 `d` |
| 无符号除 | `div s` | `edx:eax / s`，商存 `eax`，余数存 `edx` |
| 有符号除 | `idiv s` | 有符号除法，商存 `eax`，余数存 `edx` |
| 取负 | `neg d` | `d <- -d` |
| 自增 | `inc d` | `d <- d + 1` |
| 自减 | `dec d` | `d <- d - 1` |

### 1.9 `mov` 指令示例

```asm
mov eax, dword ptr [ebx]
```

把 `ebx` 所指主存地址处的 32 bit 数据复制到 `eax`。

```asm
mov dword ptr [ebx], eax
```

把 `eax` 的内容复制到 `ebx` 所指主存地址处的 32 bit 单元。

```asm
mov eax, byte ptr [ebx]
```

把 `ebx` 所指主存地址处的 8 bit 数据读入 `eax`。

```asm
mov eax, [ebx]
```

若未说明主存读写长度，通常默认 32 bit。

```asm
mov [af996h], eax
```

把 `eax` 的内容复制到地址 `af996h` 指向的主存单元；若未说明长度，默认 32 bit。

```asm
mov eax, dword ptr [ebx + 8]
mov eax, dword ptr [af996h - 12h]
```

分别表示带偏移的主存访问。

### 1.10 Intel 格式与 AT&T 格式

| 项目 | AT&T 格式 | Intel 格式 |
|---|---|---|
| 操作数顺序 | `op s, d`，源在左，目的在右 | `op d, s`，目的在左，源在右 |
| 寄存器 | 寄存器名前加 `%`，如 `%eax` | 直接写寄存器名，如 `eax` |
| 立即数 | 立即数前加 `$`，如 `$985` | 直接写数字，如 `985` |
| 主存地址 | 小括号，如 `(%eax)` | 中括号，如 `[eax]` |
| 读写长度 | 指令后缀 `b/w/l` 表示 byte/word/dword | 地址前写 `byte ptr/word ptr/dword ptr` |
| 偏移地址 | `disp(base,index,scale)` | `[base + index*scale + disp]` |

例：

```asm
# AT&T
movl 4(%ebx, %ecx, 32), %eax

# Intel
mov eax, [ebx + ecx*32 + 4]
```

### 1.11 无条件转移指令 `jmp`

无条件转移指令：

```asm
jmp <地址>
```

作用：

```text
PC <- <地址>
```

`<地址>` 可以来自：

- 常数：`jmp 128`
- 寄存器：`jmp eax`
- 主存：`jmp [999]`
- 标号：`jmp NEXT`

标号本质上是给某条指令地址起名字，便于书写与维护。

### 1.12 条件转移实现循环

用条件转移指令实现循环一般由 4 部分组成：

1. 循环前的初始化。
2. 判断是否直接跳过循环。
3. 循环主体。
4. 判断是否继续循环。

示例：求 `1 + 2 + ... + 100`。

```asm
mov eax, 0      ; eax 保存 result
mov edx, 1      ; edx 保存 i

cmp edx, 100
jg L2           ; 若 i > 100，跳过循环

L1:
add eax, edx    ; result += i
inc edx         ; i++
cmp edx, 100
jle L1          ; 若 i <= 100，继续循环

L2:
```

### 1.13 `loop` 指令实现循环

`loop` 指令隐含使用 `ecx` 作为循环计数器。

```asm
mov ecx, 500
Looptop:
    ; 循环主体
loop Looptop
```

`loop Looptop` 大致等价于：

```asm
dec ecx
cmp ecx, 0
jne Looptop
```

扩展：

```text
loopnz：当 ecx != 0 且 ZF == 0 时继续循环
loopz ：当 ecx != 0 且 ZF == 1 时继续循环
```

## 2. CPU、运算器、控制器与数据通路

### 2.1 CPU 的功能

CPU 的主要功能：

1. 指令控制：完成取指令、分析指令、执行指令，即程序的顺序控制。
2. 操作控制：一条指令通常由若干操作信号组合实现，CPU 产生并管理这些操作信号。
3. 时间控制：为各种操作提供时间顺序上的控制信号。
4. 数据加工：对数据进行算术运算和逻辑运算。
5. 中断处理：处理异常情况和特殊请求。

### 2.2 CPU 的基本结构

CPU 通常由两大部分组成：

- 运算器：完成数据加工。
- 控制器：完成指令控制、操作控制、时间控制。

控制器常见部件：

- `CU`：控制单元，分析指令并给出控制信号。
- `IR`：指令寄存器，保存当前正在执行的指令。
- `PC`：程序计数器，保存下一条指令地址，通常有自动加 1 功能。

控制器更完整的组成：

1. 程序计数器 `PC`：指出下一条指令在主存中的存放地址。
2. 指令寄存器 `IR`：保存当前正在执行的指令。
3. 指令译码器 `ID`：仅对操作码字段译码，向控制器提供操作信息。
4. 微操作信号发生器：根据 `IR`、`PSW`、时序信号生成控制信号。
5. 时序系统：产生节拍信号，由统一时钟分频得到。
6. 存储器地址寄存器 `MAR`：保存需要访问的主存地址。
7. 存储器数据寄存器 `MDR`：保存写入主存或从主存读出的信息。

### 2.3 运算器的基本组成

运算器用于实现：

- 算术运算：加、减、乘、除。
- 逻辑运算：与、或、非、异或等。

常见部件：

- `ALU`：算术逻辑单元。
- `ACC`：累加器，用于存放操作数或运算结果。
- `MQ`：乘商寄存器，在乘除运算中保存操作数或结果。
- `X`：通用操作数寄存器。

乘除相关寄存器常见分工：

| 运算 | ACC | MQ | X |
|---|---|---|---|
| 加 | 被加数、和 |  | 加数 |
| 减 | 被减数、差 |  | 减数 |
| 乘 | 乘积高位 | 乘数、乘积低位 | 被乘数 |
| 除 | 被除数、余数 | 商 | 除数 |

### 2.4 CPU 内部总线方式

CPU 内部单总线方式：将所有寄存器的输入端和输出端都连接到一条公共通路上。

优点：

- 结构简单。
- 容易实现。
- 控制逻辑相对清晰。

缺点：

- 同一时刻只能有一个部件向总线输出。
- 数据传输容易冲突。
- 性能较低。

### 2.5 专用数据通路方式

专用数据通路方式：根据指令执行过程中数据和地址的流动方向安排专用连接线路。

如果直接用导线连接多个寄存器到 ALU，会导致多个寄存器同时向 ALU 传输数据。解决方法：

- 使用多路选择器：根据控制信号选择一路输出。
- 使用三态门：控制每一路是否允许输出。

优点：

- 性能较高。
- 基本不存在总线争用。

缺点：

- 结构复杂。
- 硬件量大。
- 不易实现。

### 2.6 单总线数据通路中的典型传送

寄存器之间的数据传送：

```text
(PC) -> Bus
Bus -> MAR
```

对应控制信号：

```text
PCout 有效
MARin 有效
```

主存与 CPU 之间的数据传送，以取指为例：

```text
(PC) -> Bus -> MAR
1 -> R
MEM(MAR) -> MDR
MDR -> Bus -> IR
```

对应控制信号：

```text
PCout, MARin
读命令 R
MDRin
MDRout, IRin
```

执行加法指令时，可能的微操作序列：

```text
Ad(IR) -> Bus -> MAR
1 -> R
MEM(MAR) -> MDR
MDR -> Bus -> Y
(ACC) + (Y) -> Z
Z -> ACC
```

对应控制信号包括：

```text
MDRout, MARin
MDRout, Yin
ACCout, ALUin
Zout, ACCin
```

## 3. 指令周期与微操作时序

### 3.1 指令周期、机器周期、时钟周期

指令周期：CPU 从主存中每取出并执行一条指令所需的全部时间。

指令周期常用若干机器周期表示。机器周期又叫 CPU 周期。

一个机器周期包含若干时钟周期，也称：

- 节拍。
- `T` 周期。
- CPU 时钟周期。

时钟周期是 CPU 操作的最基本时间单位。

### 3.2 指令周期可能包含的工作周期

一条指令的指令周期可能包括：

| 工作周期 | 英文缩写 | 目的 |
|---|---|---|
| 取指周期 | `FE` | 从主存取出指令 |
| 间址周期 | `IND` | 取有效地址 |
| 执行周期 | `EX` | 执行指令功能，取操作数或写结果 |
| 中断周期 | `INT` | 保存断点并转入中断服务程序 |

流程：

```text
取指周期
  -> 判断是否有间址
    -> 若有，进入间址周期
  -> 执行周期
  -> 判断是否有中断
    -> 若有，进入中断周期
  -> 下一条指令取指
```

四个工作周期都可能有 CPU 访存操作，只是目的不同：

- 取指周期：为了取指令。
- 间址周期：为了取有效地址。
- 执行周期：为了取操作数或写结果。
- 中断周期：为了保存程序断点。

### 3.3 不同指令的指令周期长度

不同指令的周期可能不等。

示例：

- 空指令 `NOP`：通常只有取指周期。
- 加法指令 `ADD`：取指周期 + 执行周期。
- 乘法指令 `MUL`：取指周期 + 较长执行周期。
- 具有间接寻址的指令：取指周期 + 间址周期 + 执行周期。
- 带有中断响应的指令：取指周期 + 间址周期 + 执行周期 + 中断周期。

每个指令周期内机器周期数可以不等，每个机器周期内节拍数也可以不等。

### 3.4 取指周期的数据流

取指周期的任务：从主存取出当前指令，送入 `IR`，并形成下一条指令地址。

微操作：

```text
(PC) -> MAR
1 -> R
M(MAR) -> MDR
(MDR) -> IR
(PC) + 1 -> PC
```

说明：

1. 当前指令地址送入 `MAR`。
2. `CU` 发出读控制信号 `R`。
3. 主存将 `MAR` 所指内容送入 `MDR`。
4. `MDR` 中的指令送入 `IR`。
5. `PC` 自增，指向下一条指令。

### 3.5 间址周期的数据流

间址周期的任务：根据指令地址字段到主存中取出有效地址。

微操作：

```text
Ad(IR) -> MAR
1 -> R
M(MAR) -> MDR
(MDR) -> Ad(IR)
```

也可能写作：

```text
Ad(MDR) -> MAR
```

关键点：间址周期不是取操作数，而是取“操作数的有效地址”。

### 3.6 中断周期的数据流

中断：暂停当前任务，转去完成其他任务。为了能恢复当前任务，需要保存断点。

通常使用堆栈保存断点。若 `SP` 指向栈顶元素，且入栈先修改指针、后存入数据，则：

```text
(SP) - 1 -> SP
(SP) -> MAR
1 -> W
(PC) -> MDR
MDR -> M(MAR)
向量地址 -> PC
```

更抽象地说：

```text
保存 PC 旧值
装入中断服务程序入口地址
```

### 3.7 取数指令的完整工作过程

以“取数”指令为例，即从主存指定地址处取数。

取指：

```text
(PC) -> MAR
M(MAR) -> MDR
(MDR) -> IR
(PC) + 1 -> PC
```

分析：

```text
OP(IR) -> CU
```

执行：

```text
Ad(IR) -> MAR
M(MAR) -> MDR
(MDR) -> ACC
```

其中：

- `OP(IR)`：取操作码。
- `Ad(IR)`：取地址码。
- `M(MAR)`：取 `MAR` 指向主存单元中的数据。
- `(ACC)`：取 `ACC` 中的数据。

### 3.8 安排微操作时序的原则

原则一：微操作的先后顺序不得随意更改。

原则二：被控对象不同的微操作，尽量安排在一个节拍内完成。

原则三：占用时间较短的微操作，尽量安排在一个节拍内完成，并允许有先后顺序。

取指周期的典型时序安排：

| 节拍 | 微操作 |
|---|---|
| `T0` | `(PC) -> MAR` |
| `T0` | `1 -> R` |
| `T1` | `M(MAR) -> MDR` |
| `T1` | `(PC) + 1 -> PC` |
| `T2` | `MDR -> IR` |
| `T2` | `OP(IR) -> ID` |

说明：

- `M(MAR) -> MDR` 是主存读操作，用时较长，通常至少需要一个时钟周期。
- `MDR -> IR` 是 CPU 内部寄存器传送，速度快。
- `MDR -> IR` 后可以紧接着进行 `OP(IR) -> ID`，因此二者可安排在同一节拍。

### 3.9 指令执行方案

方案一：单指令周期。

- 所有指令都使用相同执行时间。
- 指令之间串行执行。
- 周期长度取决于执行时间最长的指令。
- 简单但效率低。

方案二：多指令周期。

- 不同类型指令使用不同执行步骤。
- 指令之间仍串行执行。
- 可选用不同数量时钟周期完成不同指令。
- 硬件设计更复杂。

方案三：流水线方案。

- 每个时钟周期启动一条指令。
- 多条指令并行执行，但处在不同执行阶段。
- 能提高吞吐率。
- 需要处理结构相关、数据相关、控制相关等冲突。

## 4. 控制器实现：硬布线与微程序

### 4.1 硬布线控制器

硬布线控制器用组合逻辑电路直接产生控制信号。

输入来源：

- 指令寄存器 `IR` 中的操作码。
- 节拍信号 `T0, T1, ...`。
- 工作周期触发器：`FE`、`IND`、`EX`、`INT`。
- 状态标志：来自 `PSW`、运算器、I/O、主存等。

输出：

- 各种控制信号 `C0, C1, ..., Ck`。
- 每个控制信号对应一个或多个微命令。

例：所有指令的取指周期 `T0` 节拍一定要完成：

```text
(PC) -> MAR
```

若控制信号 `C1` 控制该微操作，则：

```text
C1 = FE · T0
```

逻辑表达式就是电路的数学化描述。

硬布线控制器特点：

- 速度快。
- 适合 RISC 或指令较简单的系统。
- 控制逻辑复杂时设计和修改困难。

### 4.2 微程序控制器的设计思想

微程序控制器采用“存储程序”的思想：

```text
CPU 出厂前，将所有指令对应的微程序存入控制存储器 CM
```

关系：

- 程序：由机器指令序列组成。
- 微程序：由微指令序列组成。
- 机器指令：对程序执行步骤的描述。
- 微指令：对机器指令执行步骤的描述。
- 指令：对微命令功能的封装。
- 微指令：对微操作功能的封装。

每一种机器指令对应一个微程序。

### 4.3 微程序控制器的工作原理

主要部件：

- 控制存储器 `CM`：存放微程序。
- 控制存储器地址寄存器 `CMAR`：保存当前微指令地址。
- 微指令寄存器 / 控制存储器数据寄存器 `CMDR`：保存当前微指令。
- 地址译码：根据 `CMAR` 从 `CM` 读出微指令。
- 微地址形成部件：根据 `IR` 中操作码、寻址特征、标志等形成微程序入口或下址。
- 顺序逻辑：决定下一条微指令地址。

微指令通常包含：

```text
微指令操作码 / 操作控制字段 | 下地址字段 / 顺序控制字段
```

例：取数指令 `LDA X` 可能执行：

```text
取指周期：#0, #1, #2
间址周期：#3, #4, ..., #7
执行周期：#13, #14, #15
中断周期：#8, #9, ..., #12
```

是否进入间址周期，可能由指令地址码中的寻址特征位决定。

是否进入中断周期，可能由中断请求信号决定。

### 4.4 微指令格式

#### 水平型微指令

一条微指令能定义多个可以并行完成的微命令。

优点：

- 微程序短。
- 执行速度快。

缺点：

- 微指令长。
- 编写微程序较麻烦。

#### 垂直型微指令

一条微指令通常只定义一个微命令，由微操作码字段规定具体功能。

基本格式可能为：

```text
微操作码 μOP | 目的地址 Rd | 源地址 Rs
```

优点：

- 微指令短。
- 简单、规整。
- 便于编写微程序。

缺点：

- 微程序长。
- 执行速度慢。
- 工作效率低。

#### 混合型微指令

结合水平型和垂直型的特点，兼顾编码长度和并行能力。

补充：

- 相容性微命令：可以并行完成的微命令。
- 互斥性微命令：不允许并行完成的微命令。

### 4.5 微指令地址形成方式

常见微指令地址形成方式：

1. 微指令下地址字段指出。
   - 微指令中设置下地址字段，直接指出后继微指令地址。
   - 又称断定方式。
2. 根据机器指令操作码形成。
   - 机器指令取到 `IR` 后，由操作码经微地址形成部件生成入口地址。
3. 增量计数器法。
   - `CMAR + 1 -> CMAR`。
4. 分支转移。
   - 微指令中给出转移方式、判别条件、转移地址。
5. 通过测试网络。
6. 由硬件产生微程序入口地址。
   - 第一条微指令地址由专门硬件产生。
   - 中断周期微程序首地址也可由硬件产生。

## 5. 主存储器与外存

### 5.1 存储器芯片的基本原理

若地址线有 `n` 位，则最多可选择：

```text
2^n 个存储单元
```

若数据线宽度为 `m` 位，则存储字长为 `m` 位。

总容量：

```text
总容量 = 存储单元个数 × 存储字长
```

例：

```text
3 位地址 -> 2^3 个存储单元
8 位数据线 -> 字长 8 bit = 1 B
总容量 = 2^3 × 8 bit = 8 B
```

重要信号：

- `MAR`：地址寄存器，向地址总线输出地址。
- `MDR`：数据寄存器，通过数据总线交换数据。
- `CS` / `CE`：片选 / 使能信号，常见低电平有效。
- `WE`：写使能。

### 5.2 译码器与片选信号

CPU 先送出地址信号，地址稳定后再发出存储器请求信号 `MREQ`。

译码器可使用高位地址生成片选信号 `CS`。

注意：

- 许多芯片的片选信号低电平有效。
- CPU 可用译码器的使能端控制片选信号的生效时机。
- 片选信号不宜早于地址稳定。

### 5.3 字扩展：线选法

字扩展用于增加主存可存储的字数。

线选法：直接用高位地址线作为不同芯片的片选信号。

特点：

- `n` 条线可以产生 `n` 个片选信号。
- 线路简单。
- 地址空间利用可能不连续、不充分。

例：两个 8K×8 位芯片用高位地址线选择，若 `A14 A13` 只能为 `01` 或 `10`，则对应地址区间分别形如：

```text
01x xxxx xxxx xxxx
10x xxxx xxxx xxxx
```

### 5.4 SRAM 与 DRAM

| 特点 | SRAM | DRAM |
|---|---|---|
| 全称 | Static RAM | Dynamic RAM |
| 存储信息 | 触发器 | 电容 |
| 破坏性读出 | 否 | 是 |
| 读出后是否再生 | 不用 | 需要 |
| 运行速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 发热量 | 大 | 小 |
| 存储成本 | 高 | 低 |
| 易失性 | 易失 | 易失 |
| 是否需要刷新 | 不需要 | 需要 |
| 行列地址 | 同时送 | 分两次送 |
| 常见用途 | Cache | 主存 |

### 5.5 栅极电容与双稳态触发器

DRAM 单元通常用栅极电容存储信息：

- 读出 `1`：MOS 管接通，电容放电，数据线上产生电流。
- 读出 `0`：MOS 管接通后，数据线无电流。

SRAM 单元常用双稳态触发器保存信息：

- 状态稳定。
- 不需要刷新。
- 结构复杂，面积较大。

### 5.6 DRAM 的刷新

DRAM 中电容会漏电，因此需要周期性刷新。

常见结论：

- 刷新周期一般为 2 ms。
- 刷新以行为单位，每次刷新一行存储单元。
- 行列地址常被拆分，行地址和列地址等长。

如果有 `n` 位地址，存储单元可排列为：

```text
2^(n/2) × 2^(n/2)
```

刷新时只需要给出行地址，即可刷新该行中的所有存储单元。

### 5.7 多模块存储器

多体并行存储器：

- 每个模块有相同容量和存取速度。
- 每个模块有独立的读写控制电路、地址寄存器和数据寄存器。
- 多个模块既能并行工作，也能交叉工作。
- 适合提高连续地址访问速度。

单体多字存储器：

- 一个存储单元存放多个字。
- 总线宽度也为多个字。
- 一次并行读出多个字。
- 每次只能同时取这多个字，不能单独取其中某个字。
- 指令和数据必须连续存放才容易发挥作用。

### 5.8 SSD 固态硬盘

SSD 基于闪存 Flash Memory，属于电可擦除 ROM，即 EEPROM 的一种。

组成：

- 存储介质：多个 Flash Chip。
- 每个芯片包含多个块 `block`。
- 每个块包含多个页 `page`。
- 闪存翻译层 `FTL` 负责逻辑块号到物理页的转换。

读写特性：

- 以页为单位读 / 写，类似磁盘扇区。
- 以块为单位擦除。
- 擦干净的块中，每页通常可以写一次，读可以多次。
- 若要写的页已有数据，不能直接覆盖，需要把块内其他页复制到新擦除块，再写入新页。
- 支持随机访问，逻辑地址可快速定位到物理地址。
- 读快、写慢。

与机械硬盘相比：

- 随机访问性能高。
- 无机械寻道和旋转延迟。
- 安静、抗震、能耗低。
- 价格更高。
- 某个块擦除次数过多可能损坏。

磨损均衡：

- 动态磨损均衡：写入数据时优先选择擦除次数少的新闪存块。
- 静态磨损均衡：迁移较少变化的数据，让较新的块承担更多写任务。

## 6. Cache

### 6.1 Cache 的基本思想

Cache 利用程序访问的局部性，把近期可能再次访问的数据从主存复制到更快的小容量存储器中。

常见层次：

```text
CPU -> L1 Cache -> L2 Cache -> L3 Cache -> 主存 DRAM
```

离 CPU 越近：

- 速度越快。
- 容量越小。
- 成本越高。

离 CPU 越远：

- 速度越慢。
- 容量越大。
- 成本越低。

### 6.2 命中率与平均访问时间

设：

- `tc`：访问一次 Cache 所需时间。
- `tm`：访问一次主存所需时间。
- `H`：命中率，即 CPU 欲访问的信息已在 Cache 中的比例。
- `M`：缺失率，`M = 1 - H`。

若先访问 Cache，Cache 未命中再访问主存：

```text
t = Htc + (1 - H)(tc + tm)
```

若同时访问 Cache 和主存，Cache 命中则立即停止访问主存：

```text
t = Htc + (1 - H)tm
```

> 易错点：两个公式适用的访问策略不同，不能混用。

### 6.3 Cache 行中存储的信息

Cache 行一般包含：

```text
有效位 + 标记 + 整块数据
```

- 有效位：表示该 Cache 行内容是否有效。
- 标记：指出该 Cache 行当前对应哪个主存块。
- 数据：一个主存块的副本。

不同映射方式下，主存地址中“标记”的位数不同。

### 6.4 Cache - 主存映射方式总览

#### 全相联映射

规则：

```text
主存块可以放到 Cache 的任意位置
```

地址结构：

```text
标记 | 块内地址
```

优点：

- Cache 空间利用充分。
- 命中率较高。

缺点：

- 查找标记最慢。
- 可能需要同时比较所有 Cache 行的标记。

#### 直接映射

规则：

```text
Cache 行号 = 主存块号 mod Cache 总行数
```

地址结构：

```text
标记 | 行号 | 块内地址
```

优点：

- 对任意地址只需比较一个标记。
- 速度最快。

缺点：

- Cache 空间利用不充分。
- 容易冲突，命中率低。

#### 组相联映射

规则：

```text
所属组号 = 主存块号 mod Cache 总组数
```

主存块可以放到特定组中的任意行。

地址结构：

```text
标记 | 组号 | 块内地址
```

优点：

- 是直接映射和全相联映射的折中。
- 综合效果较好。

术语：

```text
n 路组相联映射：每 n 个 Cache 行为一组
```

### 6.5 映射方式例题

条件：

```text
主存地址空间：256 MB = 2^28 B
按字节编址
Cache：8 行
行长：64 B = 2^6 B
```

主存地址共 28 位。

块内地址：

```text
6 位
```

主存块号：

```text
28 - 6 = 22 位
```

#### 直接映射

Cache 有 8 行：

```text
行号 = 3 位
```

地址结构：

```text
19 位标记 | 3 位行号 | 6 位块内地址
```

访问步骤：

1. 根据主存块号低 3 位确定 Cache 行。
2. 比较主存块号高 19 位与 Cache 标记。
3. 若标记匹配且有效位为 1，则命中。
4. 若未命中或有效位为 0，则访问主存。

#### 全相联映射

地址结构：

```text
22 位标记 | 6 位块内地址
```

访问步骤：

1. 用地址的前 22 位与 Cache 中所有行标记比较。
2. 若某行标记匹配且有效位为 1，则命中。
3. 否则访问主存。

#### 2 路组相联映射

Cache 8 行，2 路组相联：

```text
每 2 行一组，共 4 组
组号 = 2 位
```

地址结构：

```text
20 位标记 | 2 位组号 | 6 位块内地址
```

访问步骤：

1. 根据主存块号低 2 位确定所属组。
2. 在该组内比较标记。
3. 若某行标记匹配且有效位为 1，则命中。
4. 否则访问主存。

### 6.6 多级 Cache 与写策略

现代计算机常采用多级 Cache。

常见结构：

```text
CPU -> L1 Cache -> L2 Cache -> 主存
```

写缓冲 `Write Buffer` 可缓解写操作对后续访存的阻塞。

常见策略：

- 各级 Cache 之间常采用全写法 + 非写分配法。
- Cache 与主存之间常采用写回法 + 写分配法。

基本理解：

- 全写法：写 Cache 时同时写下一层。
- 写回法：只写 Cache，替换时再写回下一层。
- 写分配：写未命中时把对应块调入 Cache。
- 非写分配：写未命中时直接写下一层，不调入 Cache。

## 7. 数值表示、标志位与加减运算

### 7.1 原码、反码、补码、移码

真值转原码：

- 正数：符号位为 0，数值位不变。
- 负数：符号位为 1，数值位不变。

原码转反码：

- 正数：不变。
- 负数：符号位不变，数值位取反。

原码转补码：

- 正数：不变。
- 负数：符号位不变，数值位取反加 1。

补码转移码：

- 符号位取反，数值位不变。
- 移码只能用于表示整数。

定点整数：

```text
符号位 | 数值部分
```

定点小数：

```text
符号位 | 小数点位置隐含 | 数值部分
```

### 7.2 逻辑门电路基础

| 逻辑运算 | 表达式 | 英文 | 特征 |
|---|---|---|---|
| 与 | `Y = A · B` | AND | 输入全 1 输出 1 |
| 或 | `Y = A + B` | OR | 输入全 0 输出 0 |
| 非 | `Y = A'` | NOT | 0 变 1，1 变 0 |
| 与非 | `Y = (A · B)'` | NAND | 输入全 1 输出 0 |
| 或非 | `Y = (A + B)'` | NOR | 输入全 0 输出 1 |
| 异或 | `Y = A ⊕ B` | XOR | 输入相异输出 1 |
| 同或 | `Y = A ⊙ B` | XNOR | 输入相同输出 1 |

### 7.3 带标志位的加减运算电路

ALU 运算结果会自动影响 `PSW` 中的标志位。

常见标志位：

| 标志 | 英文 | 含义 |
|---|---|---|
| `OF` | Overflow Flag | 溢出标志，用于判断带符号数运算是否溢出 |
| `SF` | Sign Flag | 符号标志，结果为负时置 1 |
| `ZF` | Zero Flag | 零标志，结果为 0 时置 1 |
| `CF` | Carry Flag | 进位 / 借位标志，用于判断无符号运算是否溢出 |

减法可通过加法实现：

```text
A - B = A + (-B)
```

硬件中通常：

- 被减数不变。
- 减数按位取反。
- 低位进位 `Cin = 1`。

### 7.4 无符号数加减溢出

无符号加法溢出：

```text
最高位产生进位 C = 1 -> 溢出
最高位产生进位 C = 0 -> 未溢出
```

无符号减法溢出：

减法变加法后：

```text
最高位产生进位 C = 0 -> 溢出
最高位产生进位 C = 1 -> 未溢出
```

手算判断：

```text
n bit 无符号整数范围：0 ~ 2^n - 1
超出范围即溢出
```

### 7.5 硬件如何实现比较

高级语言中的：

```c
if (a > b) {
    ...
} else {
    ...
}
```

在硬件层面通常通过减法完成比较。

`cmp a, b` 本质：

```text
a - b
```

但不保存结果，只设置标志位：

```text
OF, SF, ZF, CF
```

常见条件转移：

| 指令 | 条件 | 含义 |
|---|---|---|
| `je` | `ZF == 1` | 相等 |
| `jne` | `ZF == 0` | 不相等 |
| `jg` | `ZF == 0 && SF == OF` | 有符号大于 |
| `jge` | `SF == OF` | 有符号大于等于 |
| `jl` | `SF != OF` | 有符号小于 |
| `jle` | `SF != OF || ZF == 1` | 有符号小于等于 |

注意：无条件转移 `jmp` 不关心 `PSW` 标志位。

### 7.6 定点数的移位运算

逻辑移位：

- 逻辑左移：高位移出丢弃，低位补 0。
- 逻辑右移：低位移出丢弃，高位补 0。

算术移位：

- 算术左移：高位移出丢弃，低位补 0。
- 算术右移：低位移出丢弃，高位补符号位。

逻辑左移和算术左移的补位规则相同，但算术左移需要关注符号变化。

移位与乘除关系：

对无符号整数：

- 每逻辑左移 1 位，相当于乘以 2；若高位移出 1，则溢出。
- 每逻辑右移 1 位，相当于除以 2；若低位移出 1，则丢失精度。

对带符号整数：

- 每算术左移 1 位，相当于乘以 2；若左移前后符号位不同，则溢出。
- 每算术右移 1 位，相当于除以 2；若低位移出 1，则丢失精度。

常考：

```text
左移 r 位 <=> 乘以 2^r
右移 r 位 <=> 除以 2^r
```

## 8. 乘除运算

### 8.1 实现乘法的三种常见方式

在计算机中实现乘法运算的三种常见方式：

1. 阵列乘法器。
   - 速度最快。
   - 可以 1 个时钟内完成运算。
   - 属于快速乘法器的一种。
2. 由 `ALU + 移位器 + 寄存器 + 控制逻辑` 组成乘法电路。
   - 速度较快。
   - 通常需要多个时钟。
3. 用移位运算和加 / 减运算等效实现乘法。
   - 速度最慢。
   - 硬件最简单或可由软件完成。

### 8.2 无符号整数乘法原理

以 4 bit 无符号整数为例，乘法可通过重复加法和移位实现。

硬件寄存器：

- `X`：被乘数寄存器。
- `P`：乘积寄存器高位。
- `Y`：乘数寄存器。
- `C`：保存本轮加法产生的进位。
- `Cn`：计数器，保存剩余循环次数。

每轮操作：

1. 将乘数寄存器 `Y` 的最低位送入控制逻辑判断。
2. 若 `Y` 最低位为 1，则执行加法，结果写回 `P`，进位保存到 `C`。
3. 若 `Y` 最低位为 0，则不加。
4. 将 `[C, P, Y]` 视为整体，逻辑右移 1 位。
5. `Cn` 减 1。
6. `Cn = 0` 时结束。

最终 `P` 与 `Y` 拼接得到乘积。

### 8.3 32 位无符号乘法电路

32 位无符号乘法电路核心：

- 32 位 ALU。
- 被乘数寄存器 `X`。
- 乘积寄存器 `P`。
- 乘数寄存器 `Y`。
- 进位触发器 `C`。
- 控制逻辑与计数器 `Cn`。

若用该电路实现：

```c
unsigned int x = 985;
unsigned int y = 211;
unsigned int z = x * y;
```

过程：

- 根据 `Y` 的最低位决定是否让 ALU 执行加法。
- 控制 `[C, P, Y]` 整体逻辑右移。
- 计数器记录还需处理多少位。
- 最终结果理论上为 64 位，但若只保留低 32 位，则高 32 位不全为 0 说明发生溢出。

### 8.4 带符号整数乘法原理

补码乘法可使用 Booth 思想。

关键判断使用两位：

```text
乘数寄存器 Y 的最低位
辅助位
```

规则：

| Y 最低位 | 辅助位 | 本轮操作 |
|---|---|---|
| 0 | 0 | `+0` |
| 0 | 1 | `+[X]补` |
| 1 | 0 | `-[X]补` |
| 1 | 1 | `+0` |

每轮操作：

1. 将 `Y` 最低位和辅助位送入控制逻辑判断。
2. 根据规则决定 `+0`、`+[X]补` 或 `-[X]补`。
3. 将 `[P, Y, 辅助位]` 视为整体，算术右移 1 位。
4. `Cn` 减 1。
5. 重复 `n` 轮。

记忆：

```text
辅助位 - Y最低位 = 1  -> P + [X]补
辅助位 - Y最低位 = 0  -> P + 0
辅助位 - Y最低位 = -1 -> P - [X]补
```

### 8.5 无符号整数除法原理

除法器支持：

```text
2n bit ÷ n bit -> n bit 商 + n bit 余数
```

以 4 bit 无符号整数为例：

- `Y`：除数寄存器。
- `R`：余数寄存器。
- `Q`：被除数 / 商寄存器。
- `Cn`：计数器。

需要进行 `1 + n` 轮处理，计算 `1 + n` 位商。

第 1 轮特殊处理：

- 直接上商。
- 若第一位商为 1，说明发生商溢出，停止除法运算。
- 若第一位商为 0，说明不会发生商溢出，这一位商不保存，计数器不减。

其余 `n` 轮：

1. 先左移，空出的位用于上商。
2. 试减：`R - Y`。
3. 若结果 `>= 0`，上商 1。
4. 若结果 `< 0`，上商 0，并恢复余数。
5. `Cn` 减 1。

### 8.6 商溢出

除法器支持两种形式：

```text
2n bit ÷ n bit  -> 双精度除法
n bit  ÷ n bit  -> 单精度除法
```

最终都只保留：

```text
n bit 商 + n bit 余数
```

结论：

- 对无符号整数，双精度除法可能发生商溢出。
- 对无符号整数，单精度除法不可能发生商溢出。

例：

```text
255 / 15 = 17
1111 1111 / 1111 = 1 0001
```

若商需要超过 n bit 表示，则商溢出。

### 8.7 带符号整数补码除法

初始化：

- `n bit` 被除数需要符号扩展为 `2n bit`，放入寄存器 `[R, Q]`。
- `n bit` 除数放入寄存器 `Y`。
- 计数器 `Cn` 初值为 `n`。

特殊情况判断：

1. 若 `|被除数| < |除数|`，可直接得到：
   ```text
   商 = 0
   余数 = 被除数
   ```
2. 若除数为 0，发生除数为 0 异常。
3. 对补码单精度除法，只有“绝对值最大的负数 ÷ -1”可能发生商溢出。

循环处理：

1. 先左移，空位用于上商。
2. 根据中间余数与除数的符号组合决定做减法还是加法：
   - 同号减。
   - 异号加。
3. 根据 ALU 运算结果的新余数符号决定上商 1 还是 0。
4. 新余数与老余数相比：
   - 符号不变，上商 1。
   - 符号变化，上商 0。
5. `Cn--`，直到为 0。

结束：

```text
R 保存余数
Q 保存商
```

## 9. IEEE 754 浮点数

### 9.1 float 单精度浮点数存储

`float` 为 32 位：

```text
1 bit 符号位 | 8 bit 阶码 | 23 bit 尾数
```

存储规则：

- 符号位：`0` 表示正，`1` 表示负。
- 阶码：用移码表示，偏置值为 `127`。
- 尾数：默认存储规格化尾数，小数点前的 `1` 省略，称为隐藏位。
- 基数：2，不需要专门存储。

例：`-6.75`

十进制转二进制：

```text
6.75 = 110.11
-6.75 = -110.11
```

规格化：

```text
-110.11 = -1.1011 × 2^2
```

符号位：

```text
1
```

阶码真值：

```text
2
```

阶码存储：

```text
2 + 127 = 129 = 10000001
```

尾数：

```text
10110000000000000000000
```

最终：

```text
1 | 10000001 | 10110000000000000000000
```

### 9.2 规格化与非规格化浮点数

规格化浮点数：

- 阶码不全为 0。
- 阶码不全为 1。
- 尾数小数点前隐含 `1`。

非规格化浮点数：

- 阶码全为 0。
- 尾数不全为 0。
- 尾数小数点前隐含 `0`。
- 用于表示靠近 0 的极小数。

单精度非规格化数：

```text
正数：+2^-126 × (0.f)
负数：-2^-126 × (0.f)
```

双精度非规格化数：

```text
正数：+2^-1022 × (0.f)
负数：-2^-1022 × (0.f)
```

### 9.3 特殊值

IEEE 754 中，阶码全 0 和全 1 有特殊含义。

| 阶码 | 尾数 | 含义 |
|---|---|---|
| 全 0 | 全 0 | `+0` 或 `-0` |
| 全 0 | 非 0 | 非规格化数 |
| 全 1 | 全 0 | `+∞` 或 `-∞` |
| 全 1 | 非 0 | `NaN` |

上溢处理：

- 结果设为 `+∞` 或 `-∞`，由符号位决定。
- 设置浮点数溢出异常标志位。

下溢处理：

- 若结果落入非规格化区间，用非规格化浮点数存储。
- 若结果太小、真值逼近 0，可按机器零存储。
- 设置下溢异常标志位。

### 9.4 double 双精度浮点数

`double` 为 64 位：

```text
1 bit 符号位 | 11 bit 阶码 | 52 bit 尾数
```

阶码偏置值：

```text
1023
```

规格化正数范围大致从：

```text
+1.0 × 2^-1022
```

到：

```text
+1.111...1 × 2^1023
```

规格化负数范围对称。

非规格化最小正数：

```text
+1.0 × 2^-1074
```

非规格化最大正数：

```text
(1 - 2^-52) × 2^-1022
```

### 9.5 IEEE 754 的 4 种舍入模式

| 舍入模式 | 特点 |
|---|---|
| 就近舍入 | 默认模式，类似“四舍五入”，但遇到正好中间值时舍入到偶数 |
| 正向舍入 | 向 `+∞` 舍入；对正数进位，对负数截断 |
| 负向舍入 | 向 `-∞` 舍入；对负数进位，对正数截断 |
| 截断法 | 向 0 舍入，直接截断多余尾数，不进位 |

就近舍入常见规则：

- 若舍弃部分小于一半，直接截断。
- 若舍弃部分大于一半，末位加 1。
- 若正好等于一半，向偶数舍入，即保留部分最低位为偶数。

## 10. 函数调用栈与机器级表示

### 10.1 函数调用栈在内存中的位置

32 位系统中，进程虚拟地址空间通常为 4 GB：

```text
0x00000000 ~ 0xFFFFFFFF
```

典型布局：

- 高地址：操作系统内核区。
- 用户栈 `Stack`。
- 共享库映射区。
- 堆 `heap`，如 `malloc` 分配区域。
- 读 / 写数据区：全局变量、静态变量。
- 只读代码 / 数据：程序指令、只读数据。
- 未使用区。

用户栈通常从高地址向低地址增长：

```text
栈底：高地址
栈顶：低地址
```

### 10.2 栈帧

在函数调用栈中，一个栈帧对应一层函数调用，用于存储该函数相关信息。

一个栈帧可能包含：

1. 上一层栈帧的基地址，即 `ebp` 旧值。
2. 局部变量。
3. 空闲未使用区域。
4. 部分寄存器值。
5. 调用参数。
6. 返回地址，即 `IP` 旧值。

常见布局：

- 局部变量集中存放在栈帧底部区域。
- 调用参数集中存放在栈帧顶部区域。
- 栈帧最底部一定是上一层栈帧基址，即 `ebp` 旧值。
- 栈帧最顶部一定是返回地址，即 `IP` 旧值，当前函数栈帧除外。

编译器可能把每个栈帧大小设置为 16 B 的整数倍，因此栈帧内可能存在空闲未使用区域。

### 10.3 `push` 与 `pop`

在 x86 32 位环境中，每个栈单元常以 4 字节为单位。

`push`：

```text
先让 esp 减 4
再把数据压入 [esp]
```

`pop`：

```text
先把栈顶元素写入目的位置
再让 esp 加 4
```

`push` 的操作数可以是：

- 立即数。
- 寄存器。
- 主存地址。

`pop` 的目的位置可以是：

- 寄存器。
- 主存地址。

### 10.4 `ebp` 与 `esp`

`ebp`：

```text
指向当前栈帧的底部
```

`esp`：

```text
指向当前栈帧的顶部
```

访问栈帧数据可通过：

```asm
mov 指令 + ebp/esp 偏移
```

常见访问：

```text
[ebp - 4], [ebp - 8], ...  -> 当前函数局部变量
[ebp + 8], [ebp + 12], ... -> 上一层函数传入的参数
[ebp + 4]                 -> 返回地址
```

### 10.5 函数调用的机器级过程

调用者主要做：

1. 保存必要的寄存器。
2. 将调用参数写入当前栈帧顶部区域。
3. 执行 `call` 指令。
4. 使用返回值。
5. 恢复必要的寄存器。

被调用者主要做：

1. 保存上一层函数栈帧，设置当前函数栈帧。
2. 初始化局部变量。
3. 执行处理逻辑。
4. 通过 `eax` 寄存器向上层函数传递返回值。
5. 恢复上一层函数栈帧。
6. 执行 `ret` 指令。

### 10.6 `call` 与 `ret`

在 x86 中，程序计数器 `PC` 通常称为 `IP`，即 Instruction Pointer。

`call <函数名>` 的作用：

1. 将 `IP` 旧值压栈保存，即保存返回地址。
2. 设置 `IP` 新值，无条件转移到被调用函数第一条指令。

函数名通常作为函数起始地址的标号。

`ret` 的作用：

```text
从函数栈帧顶部找到 IP 旧值
将其出栈并恢复到 IP 寄存器
```

### 10.7 如何切换栈帧

每个函数开头通常执行：

```asm
push ebp
mov ebp, esp
```

含义：

```text
保存上一层函数的栈帧基址，即 ebp 旧值
设置当前函数的栈帧基址，即 ebp 新值
```

这也可以由 `enter` 指令完成。

每个函数 `ret` 前通常执行：

```asm
mov esp, ebp
pop ebp
```

等价于：

```asm
leave
```

含义：

```text
恢复上一层函数的栈帧
```

### 10.8 参数与返回值

传递参数：

```text
call 指令前，将调用参数写入栈帧顶部区域
```

访问上一层传入的参数：

```text
[ebp + 8], [ebp + 12], ...
```

返回值：

```text
通常通过 eax 寄存器返回
```

## 11. 高频易错点速查

1. 四地址指令格式为 `OP A1 A2 A3 A4`，含义是 `(A1) OP (A2) -> A3`，`A4` 是下址。
2. 四地址指令完成一条通常访存 4 次：取指、读 `A1`、读 `A2`、写 `A3`。
3. 变长指令取指后应是 `PC + n -> PC`，不是固定 `PC + 1`。
4. 数据寻址结构中，寻址特征决定形式地址 `A` 如何解释。
5. 间址周期取的是有效地址，不是最终操作数。
6. 取指、间址、执行、中断四个周期都可能访存，但访存目的不同。
7. `MDR -> IR` 与 `OP(IR) -> ID` 都很快，可以安排在同一节拍。
8. 硬布线控制器靠组合逻辑直接产生控制信号，微程序控制器靠控制存储器中的微指令产生控制信号。
9. 水平型微指令并行能力强、速度快，但微指令长；垂直型微指令短而规整，但微程序长、速度慢。
10. SRAM 常作 Cache，DRAM 常作主存。
11. DRAM 需要刷新，刷新一般按行进行。
12. Cache 平均访问时间要区分“先访问 Cache”还是“同时访问 Cache 与主存”。
13. 直接映射中：行号 = 主存块号 mod Cache 总行数。
14. 组相联映射中：组号 = 主存块号 mod Cache 总组数。
15. 全相联映射标记位最多，直接映射访问最快，组相联综合效果较好。
16. `OF` 判断带符号数溢出，`CF` 判断无符号数进位 / 借位。
17. `cmp a,b` 本质是做 `a-b` 并设置标志位，不保存结果。
18. 算术右移高位补符号位，逻辑右移高位补 0。
19. 无符号单精度除法 `n bit ÷ n bit` 不可能商溢出；无符号双精度除法 `2n bit ÷ n bit` 可能商溢出。
20. 补码除法中，只有“最小负数 ÷ -1”可能发生商溢出。
21. IEEE 754 规格化数阶码既不全 0 也不全 1，尾数小数点前隐含 1。
22. IEEE 754 非规格化数阶码全 0，尾数小数点前隐含 0。
23. IEEE 754 阶码全 1、尾数全 0 表示无穷；阶码全 1、尾数非 0 表示 `NaN`。
24. x86 32 位调用栈通常向低地址增长，`ebp` 指向当前栈帧底部，`esp` 指向当前栈帧顶部。
25. `call` 保存返回地址并跳转；`ret` 取出返回地址并恢复 `IP`。
26. 函数开头 `push ebp; mov ebp, esp` 建立新栈帧；函数结束 `mov esp, ebp; pop ebp` 或 `leave` 恢复旧栈帧。
27. `loop` 隐含使用 `ecx`，`loopz/loopnz` 还会结合 `ZF` 判断。

## 12. 复习建议

建议按以下顺序复习：

1. 先掌握数据通路与指令周期，因为它们是 CPU 工作过程的骨架。
2. 再学控制器，理解硬布线和微程序只是“产生控制信号”的两种实现方式。
3. 接着复习存储器和 Cache，重点练地址拆分与平均访问时间。
4. 最后复习运算方法和机器级代码，重点区分标志位、补码运算、函数栈帧。

最适合反复默写的内容：

```text
取指周期微操作
间址周期微操作
中断周期微操作
Cache 地址划分
OF/SF/ZF/CF 含义
float/double 字段划分
call/ret 与 ebp/esp 的变化
```
