# MolmoAct

**MolmoAct: Action Reasoning Models that can Reason in Space**
*Allen Institute for AI (AI2) + University of Washington | 2025 | arXiv 2508.07917*

> 这篇论文提出了一个根本性的问题：**为什么所有的 VLA 都是"观测→动作"的黑盒？** 人类在抓取杯子前，不是直接算出关节角度——我们先感知 3D 空间，再在心里描一条轨迹，最后执行动作。MolmoAct 把这个三阶段推理过程显式建模出来，让 VLA 第一次变得**可解释、可操控、可信任**。

---

## 一、核心哲学：VLA 缺了什么？

### 1.1 端到端黑盒的代价

回顾从 [[RT-2]] 到 [[π0]] 的进化：
- RT-2：图片 + 文本 → 离散动作 token
- OpenVLA：图片 + 文本 → 离散动作 token（开源的 RT-2）
- π0：图片 + 文本 → Flow Matching 连续动作（更优的动作生成）

它们有一个共同特征：**感知直接映射到动作。** 中间没有显式的"推理"环节。这带来几个严重的问题：

**1. 不可解释**：模型为什么选了那个动作？不知道。当它失败时，你只能猜测。

**2. 不可修正**：如果模型抓错了物体，你没法"告诉它"应该往左移动。你只能重新发指令，祈祷它这次做对。

**3. 泛化脆弱**：黑盒模型学到了很多"捷径"——背景颜色、光照角度——而非真正的空间理解。这就是为什么很多 VLA 在新环境中性能崩溃。

**4. 数据效率低**：因为没有结构化的中间表示，每个新任务/新环境都需要大量数据来"教"模型。

### 1.2 人类是怎么做的？

认知科学家 Barbara Tversky 的研究告诉我们：**人类的思考是 embodied（具身的）、spatial（空间的）、outside your head（在头脑之外运行的）**。在伸手拿杯子之前，我们自动完成了以下步骤：

1. **感知空间**：我离杯子有多远？杯子在什么高度？周围有障碍物吗？
2. **规划路径**：我应该从左边绕还是从右边？手要抬多高？
3. **执行动作**：按照规划好的路径移动手臂，同时持续调整。

这三个步骤在人类大脑中是**显式、有意识（或可被意识到）**的过程。你可以用手指在空气里画出你要走的路线——这就是 MolmoAct 想要让机器人做的。

### 1.3 MolmoAct 的大胆主张

> **"推理不应该被隐藏在 embedding 中，它应该是模型输出的一个显式组成部分。"**

MolmoAct 被称为 **Action Reasoning Model (ARM)**——不是"动作生成模型"，而是"动作推理模型"。"推理"这个词是关键：它意味着模型输出的是一个**推理链**，而不只是最终动作。

## 二、三阶段推理管道

MolmoAct 的输出不是简单的动作序列，而是一个**包含三个阶段的结构化推理链**：

```
输入: "把盘子放进洗碗机"
│
├─ 阶段 1: Depth Perception Tokens（深度感知）
│   100 个离散 token，编码了场景的 3D 几何信息
│   可以解码为深度图 → 这是一张有距离信息的图片
│
├─ 阶段 2: Visual Reasoning Trace（视觉推理轨迹）
│   在 2D 图像空间中的路径点序列（最多 5 个点）
│   可以叠在输入图像上 → "这就是我打算走的路线"
│
└─ 阶段 3: Action Tokens（动作解码）
    精确的机器人控制命令
    6 DoF 末端执行器位移 + 夹爪
```

这三个阶段是**自回归生成的**（一个接一个），每一阶段的输出都成为下一阶段的条件。

### 2.1 阶段一：Depth Perception Tokens——让模型"看见 3D"

**为什么需要这个阶段？**

2D 图像缺失了深度信息——一个物体看起来很大可能因为它真的很大，也可能因为它离相机近。没有深度信息，空间推理是无源之水。

**怎么实现的？**

1. 用 **Depth Anything V2**（一个强大的单目深度估计大模型）为每个训练样本生成伪深度图
2. 训练一个 **VQVAE**（Vector Quantized Variational Autoencoder）来压缩深度图：原始深度图 $H \times W \times 1$ → 100 个离散 token
3. VQVAE 的码本大小 = 128

MolmoAct 在推理时**自回归地生成这 100 个深度 token**，然后这些 token 被解码为一张完整的深度图，提供场景的 3D 几何信息。

**训练数据的关键细节：**

MolmoAct 的训练数据中包括了 "Depth Anything V2 从真实 RGB 图像估计的伪深度" 作为真实值。这意味着模型不需要真实的深度传感器——它可以**从 RGB 图像中"学习"估计深度**，然后利用自己估计的深度来做空间推理。这是一个自举（bootstrapping）的过程。

### 2.2 阶段二：Visual Reasoning Trace——让模型"表达计划"

**为什么需要这个阶段？**

传统的 VLA 把"规划"藏在模型内部的隐空间中。MolmoAct 把它外化——变成图像空间中的一条**可见的轨迹**。

**怎么实现的？**

在 2D 图像平面上定义最多 5 个关键路径点 $(x_i, y_i)$：
```
[[13, 41], [31, 52], [55, 1], [51, 2], [51, 75]]
```

这些点可以被直接渲染到输入图像上，形成一条**可视化的轨迹**。

**训练数据从哪来？**

论文使用了多种来源来生成视觉推理轨迹的"真实值"：
- **CoTracker3**：一个先进的点跟踪器，跟踪遥操作视频中机器人末端执行器的 2D 位置
- **SAM 2** 辅助的点选择
- 人工标注的轨迹点

### 2.3 阶段三：Action Decoding——从轨迹到动作

**这一步是从"图像空间中的 2D 轨迹"到"物理空间中的 3D 动作"。**

给定 (1) 深度感知 token、(2) 视觉推理轨迹、(3) 本体感觉（当前关节角度），模型生成最终的机器人动作序列。

动作表示为**离散化的 token**（256 个 bin），类似于 OpenVLA 的做法。7 维动作：
$$\text{action} = [\Delta x, \Delta y, \Delta z, \Delta \theta_x, \Delta \theta_y, \Delta \theta_z, \text{gripper}]$$

### 2.4 为什么自回归而不是并行的？

三个阶段是**自回归生成的**——阶段的输出取决于前面阶段的输出。这看起来可能降低推理速度，但有一个重要的优势：

**每一步的中间结果都是可检查、可修正的。** 如果阶段二的轨迹看起来不对，你可以**手动修改它**——这个特性是端到端黑盒 VLA 永远做不到的。

## 三、训练：预训练 + 中间训练 + 微调

### 3.1 三阶段训练流程

**预训练阶段 (Pre-training)：**

在**多模态网络数据**上做基础视觉语言预训练（类似 LLaVA 的训练）。包括图像描述、VQA、2D 指向（"图里有几个沙发？"）等任务。这建立了基础的视觉理解和语言 grounding 能力。

数据：~190K 过滤后的 OXE 机器人操作轨迹 + 互联网图像-文本数据

**中间训练阶段 (Mid-training)：**

在 **MolmoAct Dataset**（论文自建的数据集）上训练。包含：
- **10,689 条高质量机器人轨迹**
- **93 个任务**
- 多样的场景和技能

每条轨迹都标注了**完整的推理链**：深度 token + 视觉轨迹 + 动作序列。

论文发现这个中间训练阶段非常关键——使用 MolmoAct Dataset 做中间训练，平均提升了 5.5% 的通用性能。

**微调阶段 (Fine-tuning)：**

在特定的目标任务上微调，支持：
- 单臂操作
- 双手操作
- 零样本到多种未见场景

### 3.2 训练效率（惊人！）

| 模型 | 训练数据量 | GPU 小时 | SimplerEnv |
|------|-----------|---------|------------|
| GR00T N1 | 600M 样本 | **50,000** | ~60% |
| π0 | 900M 样本 | 更大 | ~68% |
| **MolmoAct** | 26.3M 样本 | **9,728** | **70.5%** |

**训练效率是 GR00T N1 的 5 倍，性能还更强。** 这颠覆了"越大越好"的叙事——MolmoAct 证明了**正确的结构化推理比盲目堆数据更有效**。

## 四、可操控性——最令人兴奋的特性

### 4.1 "手绘轨迹来纠正机器人"

这是 MolmoAct 最颠覆性的功能。因为阶段二（视觉推理轨迹）是一个**显式的、可编辑的中间表示**，用户可以：

1. 看到模型计划的轨迹叠在图像上
2. 如果轨迹不对，**直接在图像上画出正确的轨迹**
3. 模型用修正后的轨迹重新生成动作

论文实验：
- 纯语言指令纠正机器人：**42%** 成功率
- 手绘轨迹纠正机器人：**75%** 成功率

> **"说话不如画图"**——对于空间任务，视觉引导比语言指令高效得多。

### 4.2 安全性和信任

可解释性不只是学术上的 nice-to-have。在真实部署中：
- 如果你不知道模型为什么做了一个危险动作，你会信任它吗？
- 如果你不能修正模型的错误，你能安全地部署它吗？
- 如果出了事故，你能审计模型的推理过程吗？

MolmoAct 的显式推理链对这三个问题都给出了答案。

## 五、实验亮点

### 5.1 SimplerEnv (零样本 / Visual Matching)

这是目前最具挑战性的 VLA 零样本评估基准。模型需要在完全没有见过 WidowX 机器人微调数据的情况下完成任务。

MolmoAct-7B-D 达到了 **70.5%** 零样本准确率——**超越 π0 (~68%)** 和 **GR00T N1.5 (~60%)**。

### 5.2 LIBERO（仿真基准）

在 LIBERO 基准上 **86.6%** 平均成功率。特别是：
- 在长周期任务上比 ThinkAct（之前的 SOTA）高 **+6.3%**
- 这表明结构化推理在需要"理解任务序列"的长周期场景中优势最明显

### 5.3 真实世界微调

与 π0-FAST 对比：
- 单臂任务：**+10%** 任务成功率
- 双手任务：**+22.7%** 任务成功率
- 双手优势更大 → 结构化推理在协调性要求更高的双手场景中贡献更大

### 5.4 OOD 泛化

MolmoAct 的 **+23.3%** 的 OOD 泛化提升是最重要的结果之一。这证明了：**显式结构化推理带来的是真正的泛化，而非记忆。**

## 七、MolmoAct 在 VLA 演进中的位置

```
2023: RT-2 ────────→ 动作也是语言，但中间是黑盒
2024: OpenVLA ────→ 开源化了，但还是黑盒
2024: π0 ─────────→ 双系统分离了"理解"和"执行"，但中间仍不可解释
2025: MolmoAct ──→  把"推理"从隐层拉出来，变成可读可改的显式输出
```

它代表的**范式转变**：从 "VLA 是一个函数 f(image, text) → action" 到 "VLA 是一个推理者，它输出自己的思考过程，然后基于这个思考来决定行动。"

## 八、局限与挑战

1. **推理速度**：三阶段自回归生成比一步到位的端到端模型慢。但在很多应用中（尤其是对安全性要求高的场景），慢一点但可解释 > 快但不可信
2. **视觉推理轨迹的标注成本**：MolmoAct 用 CoTracker3 + SAM 2 半自动化标注，但仍有质量和规模的问题
3. **深度感知的精度**：VQVAE 压缩深度图 → 100 个 token 在复杂场景下可能不够
4. **项目很新**（2025.8）：社区生态和下游应用还不够成熟
5. **3D → 2D vs 直接 3D**：在 2D 图像空间做路径规划有其固有局限（遮挡、多视角），未来可能需要扩展到 3D 空间推理

   - 能不能让推理链变得可被语言解释（而不只是可视化）？
