# 3. RNN

核心思想：RNN 在处理序列时维护一个隐藏状态 $h_t$，用上一时刻的隐藏状态 $h_{t-1}$ 和当前输入 $x_t$ 共同生成当前输出。它不是"像人一样记忆"的精确模拟，更准确地说，是把历史信息压缩到隐藏状态中。

RNN 和传统神经网络最大的区别在于每个时间步会复用同一组参数，并把上一时刻的隐藏状态传到下一时刻。常见形式是：

$$
h_t = f(W_x x_t + W_h h_{t-1} + b)
$$

$$
y_t = g(W_y h_t)
$$

![[00_基础知识/深度学习面试题-图像算法/assets/image-24.png]]

RNN 应用场景：

- 文本生成
- 语音识别
- 机器翻译
- 生成图像描述
- 视频标记

![[00_基础知识/深度学习面试题-图像算法/assets/image-25.png]]

缺点：

- RNN 有短期记忆问题，普通 RNN 很难稳定建模很长的输入序列。
- 训练 RNN 需要投入极大的成本。
- 普通 RNN 的早期信息要经过很多次状态传递才能影响后续输出，容易被后来的输入覆盖或在反向传播中衰减。

```python
import torch.nn as nn

# 简单 RNN 示例
rnn = nn.RNN(input_size=10, hidden_size=20, num_layers=1, batch_first=True)
x = torch.randn(4, 5, 10)   # [batch, seq_len, input_size]
out, h_n = rnn(x)            # out: [4,5,20], h_n: [1,4,20]
print(f"输出形状: {out.shape}, 最后隐状态: {h_n.shape}")
```

## 3.1 RNN 训练和传统 ANN 训练异同点？

相同点：都使用链式法则和梯度下降更新参数。

不同点：

- RNN 的训练通常使用 BPTT（Backpropagation Through Time，随时间反向传播）：先把循环结构按时间展开成一个很深的前馈网络，再沿时间反向传播梯度。
- RNN 网络参数 $W_x, W_h, W_y$ 在不同时间步共享，而传统前馈网络不同层通常有不同参数。
- 对于 RNN，在使用梯度下降算法中，每一步的输出不仅依赖当前步的网络，还依赖于之前若干步的网络状态。

## 3.2 为什么 RNN 训练的时候 Loss 波动很大？

主要原因是 RNN 的梯度会沿时间链路反复相乘，序列较长时容易出现梯度消失或梯度爆炸；同时不同长度样本、截断 BPTT、隐藏状态初始化和学习率设置也会引入波动。

常见处理办法：降低学习率、使用梯度裁剪、使用 LSTM/GRU、做 LayerNorm、合理初始化隐藏状态，并尽量保证 batch 内序列长度分布不要差异过大。

## 3.3 RNN 中为什么会出现梯度消失？

梯度消失现象：BPTT 中梯度会反复乘以隐藏状态转移矩阵和激活函数导数。如果使用 tanh 或 sigmoid，导数通常小于 1；当序列很长时，多次相乘会使梯度越来越小，早期时间步几乎得不到有效更新。

实际使用中，会优先选择 tanh 函数，原因是 tanh 函数相对于 sigmoid 函数来说梯度较大，收敛速度更快且引起梯度消失更慢。

## 3.4 如何解决 RNN 中的梯度消失问题？

1. 使用门控结构，例如 LSTM、GRU，这是最常见也最有效的办法。
2. 使用梯度裁剪，主要解决梯度爆炸；虽然不能从根本上解决梯度消失，但能让训练更稳定。
3. 使用更合适的初始化、残差连接、LayerNorm、注意力机制或截断 BPTT。
4. 如果任务允许，使用 Transformer 或 CNN 等更容易并行且长程建模更强的结构。

## 3.5 CNN 与 RNN

![[00_基础知识/深度学习面试题-图像算法/assets/image-26.png]]

不同点：

1. CNN 主要在空间维度上提取局部模式，适合图像、局部窗口、局部上下文；RNN 主要在时间/序列维度上传递隐藏状态，适合顺序依赖。
2. CNN 更容易并行，训练效率高；RNN 必须按时间步递推，长序列训练较慢。
3. RNN 可以建模时间上连续状态的输出；CNN 也能处理序列，但需要通过卷积窗口、膨胀卷积或堆叠层数扩大感受野。

## 3.6 Keras 搭建 RNN

Keras 中搭建 RNN 通常使用 `SimpleRNN`、`LSTM` 或 `GRU` 层。输入形状一般是 `(timesteps, features)`，如果需要返回每个时间步的输出，设置 `return_sequences=True`；如果只需要最后一个时间步的输出，用默认设置即可。实际任务中，普通 `SimpleRNN` 较少直接使用，更多使用 `LSTM` 或 `GRU`。
