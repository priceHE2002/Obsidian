# 2. CNN

对图像（不同的数据窗口数据）和滤波矩阵做内积（逐个元素相乘再求和）的操作，在深度学习框架中通常称为卷积。严格数学意义上，多数 CNN 实现的是互相关（cross-correlation），但工程和论文里通常仍称为 convolution。

卷积神经网络由输入层、卷积层、激励层、池化层、全连接层组成。

**最左边 —— 数据输入层**，对数据做一些处理：

- 去均值（把输入数据各个维度都中心化为 0，避免数据过多偏差，影响训练效果）。
- 归一化（把所有的数据都归一到同样的范围）、PCA/白化等。均值、方差等统计量只在训练集上计算，再应用到验证集和测试集。

**中间 —— 卷积与池化**：

- CONV：卷积层，线性乘积求和。
- RELU：激励层，ReLU 是激活函数的一种。
- POOL：池化层，即取区域平均或最大。

**最右边 —— FC：全连接层**。

## 2.2 Keras 搭建 CNN

![[00_基础知识/深度学习面试题-图像算法/assets/image-13.png]]
![[00_基础知识/深度学习面试题-图像算法/assets/image-14.png]]

总结：通常情况下，一维 CNN 的架构与 CV 的二维 CNN 很相似，它将 Conv1D 层和 MaxPooling1D 层堆叠在一起，最后是一个全局池化运算或展平操作。

```python
import torch.nn as nn

# 一个简洁的 CNN 示例（PyTorch）
class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.conv = nn.Sequential(
            nn.Conv2d(3, 32, 3, padding=1),   # [B,3,32,32] -> [B,32,32,32]
            nn.ReLU(),
            nn.MaxPool2d(2),                    # -> [B,32,16,16]
            nn.Conv2d(32, 64, 3, padding=1),   # -> [B,64,16,16]
            nn.ReLU(),
            nn.MaxPool2d(2),                    # -> [B,64,8,8]
        )
        self.fc = nn.Linear(64 * 8 * 8, num_classes)

    def forward(self, x):
        x = self.conv(x)
        x = x.view(x.size(0), -1)              # flatten
        return self.fc(x)
```

RNN 在处理非常长的序列时计算代价很大，但一维 CNN 的计算代价很小，所以在 RNN 之前使用一维 CNN 作为预处理步骤是一个好主意，这样可以使序列变短，并提取出有用的表示交给 RNN 来处理。

## 2.3 经典网络分类

经典 CNN 网络可以按时间线理解：LeNet 证明卷积适合图像识别，AlexNet 推动深度 CNN 在大规模数据上成功，VGG 用小卷积核堆叠加深网络，GoogLeNet/Inception 强调多尺度和参数效率，ResNet 用残差连接解决深层退化，DenseNet 进一步强化特征复用和梯度传播。

### 2.3.1 LeNet

最早用于数字识别，针对灰度图进行训练的，输入图像大小为 $32 \times 32 \times 1$，$5 \times 5$ 卷积核，不包含输入层的情况下共有 7 层，每层都包含可训练参数。

输入的二维图像，先经过两次卷积层到池化层，再经过全连接层，最后使用 softmax 分类作为输出层。（conv1 → pool → conv2 → pool2 → 全连接层）

### 2.3.2 AlexNet

用多层小卷积叠加来替换单个的大卷积。

- 输入尺寸：$227 \times 227 \times 3$
- 卷积层：5 个
- 降采样层（池化层）：3 个
- 全连接层：2 个（不包含输出层）
- 输出层：1 个，1000 个类别

![[00_基础知识/深度学习面试题-图像算法/assets/image-17.png]]

AlexNet 比 LeNet 表现更为出色的另一个原因是它使用了 ReLU 激活函数。

#### 2.3.2.1 AlexNet 对比 LeNet 的优势？

1. AlexNet 比 LeNet 更深。
2. 用多层的小卷积来替换单个的大卷积。
3. 非线性激活函数：ReLU。
4. 防止过拟合的方法：Dropout，数据增强。
5. 大数据训练：百万级 ImageNet 图像数据。
6. 其他：GPU 实现，LRN 归一化层的使用。

### 2.3.3 VGG

构筑了 16~19 层深的卷积神经网络，VGG-16 中的 16：含有参数的有 16 个层。

VGGNet 论文中全部使用了 $3 \times 3$ 的小型卷积核和 $2 \times 2$ 的最大池化层，通过不断加深网络结构来提升性能。

- 卷积层：CONV = $3 \times 3$ filters, $s = 1$, padding = same convolution。
- 池化层：MAX_POOL = $2 \times 2$, $s = 2$。

优点：简化了卷积神经网络的结构；缺点：训练的特征数量非常大。

随着网络加深，图像的宽度和高度都在以一定的规律不断减小，每次池化后刚好缩小一半，信道数目不断增加一倍。

#### 2.3.3.1 VGG 使用 2 个 $3 \times 3$ 卷积的优势在哪里？

1. 减少网络层参数：

用两个 $3 \times 3$ 卷积比用一个 $5 \times 5$ 卷积拥有更少参数。若输入输出通道数相同，两个 $3 \times 3$ 的参数量约为 $2 \times 3 \times 3 = 18$ 份，一个 $5 \times 5$ 是 $25$ 份，前者约为后者的 $72\%$。两个 $3 \times 3$ 串联后的有效感受野相当于 $5 \times 5$。

2. 更多的非线性变换：

两个 $3 \times 3$ 卷积层拥有比一个 $5 \times 5$ 卷积层更多的非线性变换（前者可以使用两次 ReLU，后者通常只有一次），使网络对特征的表达能力更强。

#### 2.3.3.2 每层卷积是否只能用一种尺寸的卷积核？

可以，经典的神经网络一般都属于层叠式网络，每层仅用一个尺寸的卷积核，如 VGG 结构中使用了大量的 $3 \times 3$ 卷积层。

同一层特征图也可以分别使用多个不同尺寸的卷积核，以获得不同尺度的特征，再把这些特征结合起来，得到的特征往往比使用单一卷积核的要好。比如 GoogLeNet、Inception 系列的网络。

### 2.3.4 Inception（GoogLeNet）

增加了卷积神经网络的宽度，在多个不同尺寸的卷积核上进行卷积后再聚合，并使用 $1 \times 1$ 卷积降维减少参数量。

#### 2.3.4.1 Inception 结构能不能缓解梯度消失？

可以，因为 Inception 结构额外计算了两个中间 loss，防止了较深网络传播过程中的梯度消失问题。

### 2.3.5 ResNet

残差网络解决了网络退化的问题（随着网络的深度增加，准确度反而下降了）。

#### 2.3.5.1 ResNet 为什么不用 Dropout？

BN 在训练过程对每个单个样本的 forward 均引入多个样本（Batch 个）的统计信息，相当于自带一定噪音，起到正则效果，所以也就基本消除了 Dropout 的必要。（ResNet 训练 152 层深的神经网络）

#### 2.3.5.2 ResNet 网络越来越深，准确率会不会提升？

训练精度和测试精度迅速下降。

神经网络在反向传播过程中要不断地传播梯度，而当网络层数加深时，梯度在传播过程中会逐渐消失，导致无法对前面网络层的权重进行有效的调整。

#### 2.3.5.3 ResNet v1 与 ResNet v2 的区别？

通过 ResNet 残差学习单元的传播公式，发现前馈和反馈信号可以直接传输，因此捷径连接的非线性激活函数（如 ReLU）替换为 Identity Mappings。

同时，ResNet V2 在每一层中都使用了 Batch Normalization。这样处理之后，新的残差学习单元将比以前更容易训练且泛化性更强。

### 2.3.6 DenseNet

含义：前面所有层与后面层的密集连接，每一层的输入都是前面所有层输出的并集，而该层所学习的特征图也会被直接传给其后面所有层作为输入。

- 优点：缓解梯度消失问题，特征复用，加强特征传播，减少参数量。
- 缺点：内存占用高。

缓解梯度消失原因：每一层都能通过密集连接更直接地接收来自后续层和 loss 的梯度。

参数量少原因：每一层会复用前面所有层的输出，因此每层可以只产生较少的新特征图（growth rate 较小）。

#### 2.3.6.1 DenseNet 比 ResNet 好？

1. ResNet 连接方式可能会阻碍信息的流动，但是 DenseNet 每层的输出都和最终的输出直接相连，梯度可以直接从末端流到之前的所有的层。
2. DenseNet 连接有正则化的作用，可以减少过拟合。
3. DenseNet 直接连接不同层的特征图，而不是像 ResNet 一样 element-wise sum。

#### 2.3.6.2 为什么 DenseNet 比 ResNet 更耗显存？

DenseNet 的特征图像比 ResNet 大很多，导致卷积过程的计算量比 ResNet 大很多。

## 2.4 卷积层有哪些基本参数？

- **卷积核大小（Kernel Size）**：定义了卷积的局部感受野。过去常设为 5，如 LeNet-5；现在多设为 3，通过堆叠多个 $3 \times 3$ 卷积核来达到更大的有效感受野。
- **卷积核步长（Stride）**：常见设置为 1，可以覆盖所有相邻位置特征的组合；当设置为更大值时相当于对特征组合降采样。
- **填充方式（Padding）**。
- **输入通道数**：指定卷积操作时卷积核的深度。
- **输出通道数**：指定卷积核的个数。

感受野：CNN 每一层输出的特征图上的像素点在原始图像上映射的区域大小。

## 2.5 如何计算卷积层的输出的大小？

![[00_基础知识/深度学习面试题-图像算法/assets/image-18.png]]

单个空间维度的通用公式是：

$$
O = \left\lfloor \frac{I + 2P - K}{S} \right\rfloor + 1
$$

其中 $I$ 是输入大小，$K$ 是卷积核大小，$P$ 是 padding，$S$ 是 stride。如果是二维图像，高和宽分别套用一次。

## 2.6 如何计算卷积层参数数量？

卷积层参数量 = $(K_h \times K_w \times C_{in} + 1) \times C_{out}$，其中 $+1$ 是每个卷积核的 bias。如果该层不使用 bias，则去掉 $+1$。

假设输入层矩阵维度是 $96 \times 96 \times 3$，第一层卷积层使用尺寸为 $5 \times 5$、深度为 16 的过滤器（卷积核尺寸为 $5 \times 5$、卷积核数量为 16），那么这层卷积层的参数个数为 $5 \times 5 \times 3 \times 16 + 16 = 1216$ 个。

```python
import torch.nn as nn

conv = nn.Conv2d(in_channels=3, out_channels=16,
                 kernel_size=5)

# 参数量验证
total_params = sum(p.numel() for p in conv.parameters())
print(f"参数量: {total_params}")  # 5*5*3*16 + 16 = 1216
```

## 2.7 有哪些池化方法？

池化操作也叫做子采样（Subsampling）或降采样（Downsampling），往往会用在卷积层之后，通过池化来降低卷积层输出的特征维度，有效减少网络参数的同时还可以防止过拟合现象。

- 最大池化（Max Pooling）
- 平均池化（Average Pooling）

以最大池化为例，池化范围 $2 \times 2$ 和滑窗步长 $stride=2$ 相同时，每个区域只提取一次代表性特征。

## 2.8 $1 \times 1$ 卷积的作用？

1. 加入非线性函数。卷积层之后经过激励层，提升网络的表达能力。
2. 对卷积核通道数进行降维和升维，减小参数量。

## 2.9 卷积层和池化层有什么区别？

![[00_基础知识/深度学习面试题-图像算法/assets/image-19.png]]

1. 卷积层有参数，池化层没有参数。
2. 经过卷积层节点矩阵深度会改变。池化层不会改变节点矩阵的深度，但是它可以缩小节点矩阵的大小。

## 2.10 卷积核是否一定越大越好？

不一定，缺点：会导致计算量大幅增加，不利于训练更深层的模型，相应的计算性能也会降低。

卷积神经网络（VGG、GoogLeNet 等）发现通过堆叠 2 个 $3 \times 3$ 卷积核可以获得与 $5 \times 5$ 卷积核相同的感受视野，同时参数量会更少（$3 \times 3 \times 2 + 1 < 5 \times 5 \times 1 + 1$）。

优点：文本特征有时需要有较广的感受域让模型能够组合更多的特征（如词组和字符）。

卷积核的大小并没有绝对的优劣，需要视具体的应用场景而定，但是极大和极小的卷积核都是不合适的。单独的 $1 \times 1$ 极小卷积核只能用作分离卷积而不能对输入的原始特征进行有效的组合，极大的卷积核通常会组合过多的无意义特征从而浪费了大量的计算资源。

## 2.11 卷积在图像中有什么直观作用？

用来提取图像的特征，但不同层次的卷积操作提取到的特征类型是不相同的：

- 浅层卷积：边缘特征
- 中层卷积：局部特征
- 深层卷积：全局特征

## 2.12 CNN 中空洞卷积的作用是什么？

空洞卷积也叫扩张卷积，在保持参数个数不变的情况下增大了卷积核的感受野，同时它可以保证输出的特征映射的大小保持不变。一个扩张率为 2 的 $3 \times 3$ 卷积核，感受野与 $5 \times 5$ 的卷积核相同，但参数数量仅为 9 个。

```python
# 普通 3x3 卷积 → 感受野 3x3
conv_normal = nn.Conv2d(64, 64, 3, padding=1)

# 空洞卷积 dilation=2 → 感受野 5x5，参数量仍为 3x3
conv_dilated = nn.Conv2d(64, 64, 3, padding=2, dilation=2)

print(f"普通卷积参数量: {sum(p.numel() for p in conv_normal.parameters())}")
print(f"空洞卷积参数量: {sum(p.numel() for p in conv_dilated.parameters())}")
```

## 2.13 怎样才能减少卷积层参数量？

1. **使用堆叠小卷积核代替大卷积核**：VGG 网络中 2 个 $3 \times 3$ 的卷积核可以代替 1 个 $5 \times 5$ 的卷积核。
2. **使用分离卷积操作**：将原本 $K \times K \times C$ 的卷积操作分离为 $K \times K \times 1$ 和 $1 \times 1 \times C$ 的两部分操作。
3. **添加 $1 \times 1$ 的卷积操作**：与分离卷积类似，但是通道数可变，在 $K \times K \times C_1$ 卷积前添加 $1 \times 1 \times C_2$ 的卷积核（满足 $C_2 < C_1$）。
4. **在卷积层前使用池化操作**：池化可以降低卷积层的输入特征维度。

## 2.14 在进行卷积操作时，必须同时考虑通道和区域吗？

1. 标准卷积同时考虑通道和区域。
2. 通道分离（深度分离）卷积网络（Xception 网络）：首先对每一个通道进行各自的卷积操作，有多少个通道就有多少个过滤器。得到新的通道特征矩阵之后，再对这批新通道特征进行标准的 $1 \times 1$ 跨通道卷积操作。

## 2.15 采用宽卷积、窄卷积的好处有什么？

宽卷积、窄卷积其实是一种填充方式。

- **宽卷积（SAME 填充）**：对卷积核不满足整除条件的输入特征进行补全，以使卷积层的输出维度保持与输入特征维度一致。
- **窄卷积（VALID 填充）**：不进行任何填充，在输入特征边缘位置若不足以进行卷积操作，则对边缘信息进行舍弃，因此在步长为 1 的情况下该填充方式的卷积层输出特征维度可能会略小于输入特征的维度。

## 2.16 介绍反卷积（转置卷积）

![[00_基础知识/深度学习面试题-图像算法/assets/image-20.png]]

正向传播时乘以卷积核的转置矩阵，反向传播时乘以卷积核矩阵，由卷积输出结果近似重构输入数据，上采样。

输入：$2 \times 2$，卷积核：$4 \times 4$，滑动步长：3，输出：$7 \times 7$。

过程如下：

1. 输入图片每个像素进行一次 full convolution，根据 full convolution 大小计算可以知道每个像素的卷积后大小为 $1+4-1=4$，即 $4 \times 4$ 大小的特征图。输入有 4 个像素，所以会得到 4 个 $4 \times 4$ 的特征图。
2. 将 4 个特征图进行步长为 3 的相加；输出的位置和输入的位置相同。步长为 3 是指每隔 3 个像素进行相加，重叠部分进行相加，即输出的第 1 行第 4 列是由红色特征图的第一行第四列与绿色特征图的第一行第一列相加得到，其他类推。

可以看出反卷积的大小是由卷积核大小与滑动步长决定，设 $in$ 是输入大小，$k$ 是卷积核大小，$s$ 是滑动步长，$out$ 是输出大小，得到 $out = (in - 1) \times s + k$。上图过程就是 $(2 - 1) \times 3 + 4 = 7$。

```python
# 转置卷积示例
x = torch.randn(1, 64, 2, 2)                  # 输入 2x2
conv_transpose = nn.ConvTranspose2d(64, 3, kernel_size=4, stride=3)
out = conv_transpose(x)                        # 输出 7x7
print(out.shape)  # torch.Size([1, 3, 7, 7])
```

## 2.17 如何提高卷积神经网络的泛化能力？

![[00_基础知识/深度学习面试题-图像算法/assets/image-21.png]]

常见方法：

1. **数据增强**：随机裁剪、翻转、颜色扰动、旋转、MixUp、CutMix。
2. **正则化**：weight decay、Dropout、Label Smoothing、Early Stopping。
3. **归一化和结构设计**：BatchNorm、残差连接、更合理的网络深度和宽度。
4. **预训练和迁移学习**：使用 ImageNet 等大数据集预训练模型，再在目标数据集上微调。
5. **减少数据泄漏**：训练、验证、测试严格划分，归一化统计量只从训练集计算。

## 2.18 卷积神经网络在 NLP 与 CV 领域应用的区别？

自然语言处理对一维信号（词序列）做操作，输入数据通常是离散取值（例如表示一个单词或字母通常表示为词典中的 one-hot 向量）。

计算机视觉则是对二维（图像）或三维（视频流）信号做操作。输入数据是连续取值（比如归一化到 0、1 之间的灰度值）。

## 2.19 全连接、局部连接、全卷积与局部卷积的区别？

![[00_基础知识/深度学习面试题-图像算法/assets/image-22.png]]

可以从"连接范围"和"权值是否共享"两个维度记：

| 类型 | 连接范围 | 权值共享 | 特点 |
|---|---|---|---|
| 全连接 | 每个输出连接所有输入 | 否 | 参数多，捕获全局信息 |
| 局部连接 | 每个输出只连接局部区域 | 否 | 保留局部性，但参数仍较多 |
| 卷积/全卷积 | 每个输出连接局部区域 | 是 | 参数少，平移等变，适合图像 |
| 局部卷积 | 局部区域卷积但不同位置不共享权重 | 否 | 更灵活，但参数和过拟合风险更高 |

## 2.20 卷积层和全连接层的区别？

1. 卷积层是局部连接，所以提取的是局部信息；全连接层是全局连接，所以提取的是全局信息。
2. 当卷积层的局部连接是全局连接时，全连接层是卷积层的特例。

## 2.21 Max Pooling 如何工作？还有其他池化技术吗？

1. Max Pooling：选取滑动窗口的最大值。
2. Average Pooling：平均滑动窗口的所有值。
3. Global Average Pooling：平均每页特征图的所有值。

## 2.22 卷积神经网络的优点？为什么用小卷积核？

多个小的卷积核叠加使用要远比一个大的卷积核单独使用效果要好的多。

1. **局部连接**：每个神经元不再和上一层的所有神经元相连，而只和一小部分神经元相连。这样就减少了很多参数。
2. **权值共享**：一组连接可以共享同一个权重，而不是每个连接有一个不同的权重，这样又减少了很多参数。
3. **下采样**：Pooling 层利用图像局部相关性的原理，对图像进行子抽样，可以减少数据处理量同时保留有用信息。通过去掉 Feature Map 中不重要的样本，进一步减少参数数量。

## 2.23 CNN 拆成 $3 \times 1$ 和 $1 \times 3$ 的优点？

为了压缩模型参数量（这里参数由 $3 \times 3 = 9$ 降低到 $1 \times 3 + 3 \times 1 = 6$），但是计算量基本没变（乘加数目没变）。

## 2.24 BN、LN、IN、GN 和 SN 的区别？

将输入的 feature map shape 记为 $[N, C, H, W]$，其中 $N$ 表示 batch size，即 $N$ 个样本；$C$ 表示通道数；$H$、$W$ 分别表示特征图的高度、宽度。

![[00_基础知识/深度学习面试题-图像算法/assets/image-23.png]]

几种归一化的核心差别在于"在哪些维度上计算均值和方差"：

| 方法 | 归一化维度 | 特点 | 常见场景 |
|---|---|---|---|
| BN（BatchNorm） | 对同一通道，在 $N,H,W$ 上统计 | 依赖 batch 统计，小 batch 时不稳定 | CNN 分类/检测常用 |
| LN（LayerNorm） | 对单个样本，在 $C,H,W$ 或特征维上统计 | 不依赖 batch，序列模型稳定 | RNN、Transformer |
| IN（InstanceNorm） | 对单个样本、单个通道，在 $H,W$ 上统计 | 去除实例风格信息 | 风格迁移、生成模型 |
| GN（GroupNorm） | 对单个样本，把通道分组后在组内统计 | 小 batch 稳定 | 检测、分割、小 batch 训练 |
| SN（SwitchableNorm） | 学习 BN/LN/IN 的加权组合 | 自动选择统计方式，但更复杂 | 一些视觉任务实验中使用 |

```python
import torch.nn as nn

# 各归一化层用法对比
x = torch.randn(2, 64, 32, 32)  # [N, C, H, W]

bn = nn.BatchNorm2d(64)         # BN: 在 N,H,W 维度统计
ln = nn.LayerNorm([64, 32, 32]) # LN: 在 C,H,W 维度统计
in_ = nn.InstanceNorm2d(64)     # IN: 每样本每通道在 H,W 统计
gn = nn.GroupNorm(num_groups=8, num_channels=64)  # GN: 分组后在组内统计
```

**补充问答**

**为什么需要卷积？不能使用全连接层吗？**

可以使用全连接层，但图像输入维度很高，全连接层参数量巨大，也忽略了图像的局部结构。卷积通过局部连接和权值共享显著减少参数，并能利用局部纹理、边缘、形状等空间结构；同时卷积对平移具有等变性，更适合视觉任务。

**为什么降采样使用 Max Pooling，而分类使用 Average Pooling？**

中间层降采样常用 Max Pooling，是因为最大值更强调局部区域内最显著的特征，有利于保留边缘、纹理等响应。分类头常用 Global Average Pooling，是因为它把每个通道的空间响应做全局汇总，参数少、过拟合风险低，也更容易解释为"某类特征整体出现的强度"。

**CNN 是否抗旋转？如果旋转图像，CNN 的预测会怎样？**

普通 CNN 天然更擅长处理平移，不天然具备旋转不变性。图像旋转后，卷积核看到的局部模式方向发生变化，预测可能下降。常见解决方法是旋转数据增强、使用更强的预训练模型、加入空间变换网络（STN）或使用具备旋转等变性的网络结构。

**什么是数据增强？为什么需要它们？你知道哪些增强？**

数据增强是在训练阶段对样本做保持语义不变或近似不变的随机变换，用来扩大数据分布、减少过拟合、提高鲁棒性。图像中常见增强包括随机裁剪、水平翻转、颜色扰动、旋转、仿射变换、模糊、噪声、Random Erasing、MixUp、CutMix、Mosaic 等。

**如何选择要使用的增强？**

原则是增强后的样本仍然符合任务语义。分类任务可以使用较强的裁剪、颜色扰动、MixUp/CutMix；检测和分割要同步变换标注框或 mask；医学、遥感等场景要谨慎使用会改变语义的旋转或颜色增强。最终以验证集效果和错误案例分析为准。

**什么是迁移学习？它是如何工作的？**

迁移学习是把在大数据集上学到的通用特征迁移到目标任务上。常见做法是加载预训练 backbone，替换任务相关的分类头/检测头，先训练新头部，再视数据量和领域差异决定是否解冻部分或全部 backbone 微调。底层特征通常更通用，高层特征更任务相关。

**什么是目标检测？你知道有哪些框架吗？**

目标检测需要同时回答"图中有哪些类别"和"它们在哪里"，输出类别和边界框。常见框架包括两阶段方法 Faster R-CNN、Mask R-CNN；一阶段方法 YOLO、SSD、RetinaNet、FCOS；Transformer 系列 DETR、Deformable DETR 等。

**什么是对象分割？你知道有哪些框架吗？**

对象分割比检测更细，需要输出像素级区域。语义分割给每个像素分配类别，不区分同类实例；实例分割还要区分同类不同对象；全景分割同时处理 stuff 和 thing。常见方法包括 FCN、U-Net、DeepLab、PSPNet、Mask R-CNN、Mask2Former、SAM 等。
