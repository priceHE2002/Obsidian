# π0

**π0: A Vision-Language-Action Flow Model for General Robot Control**
*Physical Intelligence | RSS 2025 | arXiv 2410.24164*

> **当前公认最强的开源 VLA。** 一切在它之前的 VLA 用离散 token + 自回归做动作生成（RT-2, OpenVLA）；π0 第一个把 Flow Matching 引入 VLA，用连续生成 + VLM 语义理解实现了 50Hz 的高频精控。它的双系统架构（VLM 大脑 + Action Expert 小脑）和预训练/后训练分离的范式，成为后续所有 VLA 的参考设计。

---

## 一、背景：为什么 VLA 需要新一代架构？

### 1.1 现有 VLA 的三大痛点

**痛点 1：动作生成的表达能力不够**

RT-2 和 OpenVLA 把连续动作离散化到 256 个 bin，然后用自回归方式逐 token 生成。这有本质局限：
- 离散化信息损失：256 个 bin 对精细操作可能不够
- 自回归生成慢：逐一生成每个 token
- 无法表达多模态分布（虽然在离散空间中近似可行，但不优雅）

**痛点 2：缺少专门的灵巧操作能力**

VLM 擅长语义理解和推理，但它们没有经过专门的灵巧操作训练。当需要"把鸡蛋轻轻放进篮子里"时，仅有语义理解是不够的——还需要精确的力控和轨迹。

**痛点 3：缺少系统的训练方法**

当你有来自 7 种不同机器人的多样化数据时，如何训练一个统一的策略？数据格式、动作空间、控制频率各不相同——仅仅是格式对齐就有巨大的工程量。

### 1.2 π0 的应对

π0 对每个痛点都有针对性的创新：
1. **Flow Matching** → 连续动作空间，更优的多模态建模
2. **双系统架构** → VLM 负责语义，Action Expert 负责灵巧
3. **预训练/后训练分离** → 大量多样化数据 + 少量高质量数据的系统方案

## 二、核心技术：Flow Matching

### 2.1 从 Diffusion 到 Flow Matching

[[Diffusion Policy]] 使用了 DDPM（Denoising Diffusion Probabilistic Model）。其去噪路径是随机的——每一步往数据分布走一小步，同时加一点噪声。虽然最终效果很好，但路径是"蜿蜒"的。

**Flow Matching 的思想更直接**：学习一个**确定性的最优传输路径**（Optimal Transport），从纯噪声**直线**走向目标数据。

数学上，Flow Matching 定义一个时变向量场 $v_t(x)$，描述概率质量如何从先验 $p_0$（高斯噪声）流向目标分布 $p_1$（数据分布）：

$$\frac{dx}{dt} = v_t(x_t)$$

训练目标就是学习这个向量场：

$$\mathcal{L}_{\text{FM}} = \mathbb{E}_{t, x_0, x_1} \left[ \| v_t^\theta(x_t) - (x_1 - x_0) \|^2 \right]$$

其中 $x_t = (1-t)x_0 + t x_1$ 是噪声和数据之间的线性插值。

**核心优势 vs Diffusion：**
- 确定性路径 → 更少推理步数（$\sim 10$ 步 vs Diffusion 的 $\sim 100$ 步训练 / $\sim 10$ 步推理）
- 最优传输 → 路径更直接，不会"绕路"
- 连续建模 → 天然适合机器人连续动作空间

### 2.2 为什么 Flow Matching 特别适合机器人？

1. **动作空间是连续的**（关节角度、末端位移），Flow Matching 输出连续向量不需要离散化
2. **真实动作分布通常不是高斯**——它可能是多峰的/非对称的/尖峰的，Flow Matching 理论上可以建模任何分布
3. **对高频控制友好**——可以产出 50Hz 的平滑轨迹

## 三、双系统架构：为什么是 MoE？

### 3.1 设计动机

如果你让 VLM 直接输出连续动作，有两个问题：
1. VLM 的参数量居大（10B+），逐 step 推理太慢
2. VLM 的设计目标不是生成平滑的动作轨迹——它擅长的是"理解场景"和"推理语义"

### 3.2 架构方案

```
┌─────────────────┐      ┌──────────────────┐
│    VLM (PaliGemma)│ ──→ │ Action Expert (DiT)│
│    "大脑"         │ 交叉  │    "小脑"          │
│    语义理解+推理  │ 注意力│    300M参数       │
│    3B            │      │    动作生成50Hz    │
└─────────────────┘      └──────────────────┘
```

**VLM 的工作**：输入为图像 + 语言指令，输出为对场景的语义理解（embedding）

**Action Expert 的工作**：输入为 VLM 的语义 embedding + 本体感觉信息，输出为**连续动作序列**（chunk size = 50，即 1 秒的动作）。使用 DiT (Diffusion Transformer) 架构，通过 Flow Matching 训练。

**两者通过交叉注意力连接**：Action Expert 的每一层都可以"查询"VLM 对当前场景的理解。这样 VLM 的语义知识直接影响动作生成。

### 3.3 与直接"微调 VLM"的区别

RT-2/OpenVLA 直接微调 VLM 的所有参数来输出动作。π0 则是新增一个 Action Expert，VLM 的参数可以保持更稳定。

**优点**：
- Action Expert 更小（300M），更快
- VLM 的语义知识不会被"稀释"（因为 VLM 主要是冻住的或小学习率微调）
- 可以灵活替换 VLM（PaliGemma → 更强的新 VLM），Action Expert 架构不变

## 四、预训练/后训练分离——借鉴 LLM 的成功经验

这是 π0 最有远见的设计之一。

### 4.1 两个阶段的性质对比

| | 预训练 (Pre-training) | 后训练 (Post-training) |
|---|---|---|
| **数据量** | 极大（10,000+ 小时机器人数据）| 小（精选高质量数据）|
| **数据来源** | 7 种机器人，68 种任务 | 目标任务特定的精调数据 |
| **数据质量** | 混合质量——有好的演示也有粗糙的 | 高质量、一致的演示 |
| **学什么** | 广泛的物理世界知识、恢复行为、纠错能力 | 高效、精准、优美的行为模式 |
| **类比** | LLM 的 web-scale pre-training | LLM 的 RLHF/指令微调 |

### 4.2 为什么两个阶段缺一不可？

论文给出的核心直觉：**"高质量数据里没有恢复行为"。**

如果你的训练数据全是专家演示的完美轨迹，模型会学到"世界是确定的"——它不会知道如果抓取失败了该怎么办。但如果你在预训练阶段混入了大量"不完美"的数据（笨拙的演示、纠正动作、不同风格的执行），模型就学会了**如何应对不确定性**。

后训练阶段则让模型**精细化**：在已经知道"世界不确定"的基础上，学习在目标场景中的最优行为。

## 五、关键实验与结果

### 5.1 多任务零样本

π0 在预训练后直接可以执行多种任务，无需任务特定的微调。在 SimplerEnv 基准上达到 **~68%** 综合成功率（推出时的 SOTA）。

### 5.2 微调后的能力

微调后展现出令人印象深刻的灵巧操作能力：
- **叠多件衣服**：取衣服 → 放到篮子里 → 推篮子到折叠台 → 逐一折叠
- **组装盒子**：灵巧地折叠纸板
- **清理桌面**：识别并拾取多件物品
- 这些都是长周期（>10 分钟）、多阶段、高精度的任务

### 5.3 与高层 VLM 策略的协作

π0 可以接受来自上层策略的**中间语言指令**，形成"高层策略 → π0 → 低层执行"的层级架构。高层 VLM 负责规划和推理（"先拿这个，再拿那个"），π0 负责执行每个子任务。

## 七、硬件约束

❌ **4070 Ti Super 16GB 无法微调 π0**
- LoRA 微调需要 >22.5 GB
- 全量微调需要 >70 GB
- PyTorch 版本（2025.9 新增）的显存需求可能略低于 JAX 版本，但差距不大

✅ **推理可行**
- 推理仅需 >8 GB
- 可以下载 π0.5 预训练权重做零样本评估

💡 **你应该做什么**：如果你不能微调 π0，现阶段的最佳策略是：
1. 先用 [[SmolVLA]] 在 16GB 上实验 VLA 训练
2. 理解 π0 的架构思想（Flow Matching + 双系统 + 预训练/后训练）
3. 等社区发布量化后的 π0（或者自己试试把 π0 量化到 16GB 可微调）

## 八、核心启示

1. **Flow Matching > 离散 token 自回归**——连续生成更优雅、更快、表达能力更强。这是 VLA 动作生成的未来方向
2. **VLM + Action Expert 的分离设计是正解**——让 VLM 做它擅长的（语义理解），让专门的模块做它擅长的（动作生成）
3. **预训练/后训练分离可能比架构更重要**——这决定了模型是否能同时学会"应对不确定性"和"精准执行"
4. **开源生态的力量**：π0 家族的开源让整个社区可以基于最强模型做研究
