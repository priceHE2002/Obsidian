# Motus

**Motus: A Unified Latent Action World Model**
*清华TSAIL + 北大 + Horizon Robotics | CVPR 2026 | arXiv 2512.13030*

> **"你们 VLA 只会条件反射。真正的智能应该能想象——如果我这样做，世界会变成什么样？"** Motus 是第一个将 VLA、世界模型、逆动力学、视频生成和视频-动作联合预测五大任务统一在一个框架中的大一统模型。在 RoboTwin 2.0 上达到 88%，比 π0.5 高 45%。

---

## 一、为什么需要"世界模型"？

### 1.1 纯 VLA 的本质局限

VLA 本质上是一个条件反射系统：**当前状态 → 下一步动作。** 它不"理解"物理规律——只是记住"在这种情况应该这样做"。

这就导致了众所周知的泛化问题：
- 对未见过的环境变化极度敏感（换个背景颜色就崩溃）
- 无法做长期规划（因为没有"预见"能力）
- 数据效率极低（每个新任务、每个新场景都需要大量数据）

### 1.2 Motus 的宏大愿景

Motus 的核心赌注：**预测未来状态的能力（世界模型）会让机器人策略更鲁棒、更通用。**

具体来说，Motus 统一了以下五种范式：
1. **VLA** — 给定当前观测，预测动作
2. **世界模型** — 给定当前状态 + 动作，预测未来状态
3. **逆动力学 (IDM)** — 给定当前和未来状态，反推动作
4. **视频生成 (VGM)** — 无条件或条件生成未来视频
5. **视频-动作联合预测** — 同时预测未来帧和动作

## 二、MoT 架构（Mixture-of-Transformer）

Motus 的核心架构创新是 **MoT (Mixture-of-Transformer)**——将三个专家模型融合在共享的注意力层中：

### 2.1 三个专家

| 专家 | 基础模型 | 参数量 | 角色 |
|------|---------|--------|------|
| **理解专家 (大脑)** | Qwen3-VL-2B | 2B | 场景理解 + 语言指令解析 |
| **视频生成专家 (想象力)** | Wan 2.2 | 5B | 推演未来画面 |
| **动作专家 (小脑)** | 自建 Transformer | ~300M | 运动控制输出 |

### 2.2 Tri-model Joint Attention

三个专家不是独立运行的——它们在**共享注意力层**中交换信息：

```
理解专家的 embedding ──┐
                       ├── Cross-Attention ──→ 动作专家
视频专家的 embedding ──┘
```

"大脑"理解场景 → "想象力"推演未来 → "小脑"决定动作，三者形成闭环。

### 2.3 为什么 MoT 比"一个大模型"更好？

- **模块化**：每个专家可以独立地使用不同的基础模型和预训练数据
- **可解释**：三个系统的功能和交互清晰
- **高效**：每个专家专注于自己的任务，不需要一个超大模型面面俱到
- **可扩展**：可以替换更强的视觉基础模型而不影响其他专家

## 三、关键创新：潜在动作（Latent Action）

### 3.1 问题：互联网视频没有动作标签

世界模型需要大量视频数据来学习"如果做 X 会发生 Y"。但互联网上的视频**没有动作标签**——你看到一个人拿起杯子，但你看不到关节角度。

### 3.2 解决方案：光流 → 潜在动作

Motus 用一个巧妙的设计绕过了这个问题：

1. **提取光流**：用现成的光流估计器从视频中提取逐像素运动信息
2. **压缩**：通过 DC-AE 将高维光流压缩为 **14 维潜在动作向量**
3. **对齐**：用少量的机器人动作数据训练一个映射，将 14 维潜在动作与真实机器人动作空间对齐

这样，Motus 可以从**海量无标签人类视频**中学习通用的运动先验——不需要标注，不需要遥操作。

### 3.3 数据效率的惊人提升

| | π0.5 | Motus |
|---|---|---|
| 达到同等性能所需数据 | 基准 | **1/13.55** |
| 数据来源 | 仅机器人数据 | 机器人数据 + 人类视频（无标签）|

## 四、三阶段训练 + 六层数据金字塔

### 4.1 训练阶段

| 阶段 | 目标 | 数据 |
|------|------|------|
| **Stage 1** | 学习视觉动力学（"如果我这样做，会看到什么"）| 多机器人轨迹 + 人类视频 |
| **Stage 2** | 学习动作表示（视觉运动特征 ↔ 机器人动作）| 视频 + 语言 + 潜在动作 |
| **Stage 3** | 目标机器人专业化 | 目标机器人的真机数据（少量）|

### 4.2 六层数据金字塔

从底部到顶部（质量和成本递增）：
1. 无标签互联网视频（海量，零成本）
2. 标注了光流的人类操作视频
3. 仿真机器人数据
4. 低质遥操作数据
5. 高质量遥操作数据
6. 专家级精细操作演示

## 六、"VLA is Dead, Long Live WAM"

Motus 是 2026 年初"VLA vs WAM"大辩论中的关键作品。

**这场辩论的本质：**

| | VLA | WAM |
|---|---|---|
| 训练逻辑 | 模仿遥操作演示（"条件反射"）| 预测世界如何演化（"物理理解"）|
| 泛化来源 | 大量多样化演示数据 | 物理先验 + 对未来状态的预测能力 |
| 数据来源 | 昂贵的遥操作 | 互联网视频 + 仿真 + 少量遥操作 |
| 长周期任务 | 弱（2-3步）| 强（10+步）|
| 推理 | 快（轻量映射）| 慢（但 FastWAM 证明可以很快）|

NVIDIA 的 Jim Fan 在 2026 年 4 月 Sequoia 大会上公开宣称"VLA is dead, WAM should rise"。但这个论断仍有争议——VLA 在精细控制、工程成熟度、低计算需求方面仍有优势。

## 七、与你的研究的关系

❌ Motus 的训练需要大量 GPU，不适合你的 16GB 单卡。

但研究方向上的启示是巨大的：

1. **世界模型是 VLA 的下一个进化方向**——无论你最终用什么架构，思考"怎样让模型学会预测未来"是方向
2. **光流 → 潜在动作的桥接方案**可以让你利用无标签视频数据
3. **MoT 的模块化设计**值得在更小的规模上尝试（比如在你的 LeRobot + SmolVLA pipeline 中加入一个小的视频预测模块）
