# RT-2

**RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control**
*Google DeepMind | CoRL 2023 | arXiv 2307.15818*

> 如果说 [[Open X-Embodiment & RT-X]] 是 VLA 的"数据地基"，那 RT-2 就是 VLA 的"概念发源地"。这篇论文首次提出了 "Vision-Language-Action Model" 这个术语，并证明了：**把机器人动作当成文本 token 来训练，可以让 VLM 直接做低层机器人控制，而且会出现令人惊叹的涌现能力。**

---

## 一、核心思想：动作即语言

### 1.1 背景：为什么传统方法不够好？

在 RT-2 之前，使用 LLM/VLM 做机器人的方法大致分为两类：

**方法 A：LLM 做高层规划**
- LLM 把自然语言指令分解为子任务序列："先走到桌子前，然后拿起杯子，然后走到水槽前"
- 但每个子任务由独立的控制器执行（手写规则、独立训练的抓取网络等）
- **问题**：底层控制器没有受益于 LLM 的语义知识。LLM 知道"能量饮料是什么"，但抓取控制器只看到物体的几何形状

**方法 B：VLM 做视觉特征提取**
- 用 CLIP 等 VLM 的视觉编码器为机器人策略提供特征
- 用一个单独的神经网络（如 RT-1 的 Transformer）将特征映射为动作
- **问题**：VLM 的语义理解没有直接流入动作生成。VLM 看到"这是一个苹果"，但策略网络只收到一个特征向量，不知道"苹果"这个概念

RT-2 的洞察是：**为什么不直接让 VLM 输出动作？** VLM 已经学会了"看图说话"——给定图片，输出文本。如果我们把动作也变成一种"语言"（文本 token），VLM 就天然可以输出动作。

### 1.2 动作 token 化方案

这是 RT-2 最巧妙的设计。机器人动作空间是 7 维的：

$$\text{action} = [\Delta x, \Delta y, \Delta z, \Delta \theta_x, \Delta \theta_y, \Delta \theta_z, \text{gripper}] + \text{terminate}$$

连续值被离散化到 256 个 bin 中。因此一个完整动作被表示为：

```
"1 128 91 241 5 101 127"
```

这个字符串可以直接出现在 VLM 的训练数据中，**和自然语言混在一起**：

```
训练样本:
  Q: "What action should the robot take to pick up the coke can?" A: "1 128 91 241 5 101 127"
  Q: "What is happening in the image?" A: "A grey donkey walks down the street."
  Q: "Que puis-je faire avec ces objets?" A: "Faire cuire un gâteau."
```

这三条数据在 VLM 训练中**完全平等**——没有任何架构修改，没有任何新参数。

**token 映射的技术细节：**
- **PaLI-X 骨干**：整数 0-999 都有自己的专用 token，所以直接映射 `bin_0 → token_0`, `bin_128 → token_128`
- **PaLM-E 骨干**：没有这种便利的整数 token 化，所以直接**覆写 256 个最不常用的 token** 作为动作 token

## 二、关键训练技巧

### 2.1 Co-Fine-Tuning（联合微调）

这是 RT-2 最重要的训练策略。**不是"把 VLM 冻住然后加一个动作头"**，也不是 **"扔掉所有 VLM 数据只用机器人数据微调"**。而是在每个 batch 中**同时包含原始 VLM 数据和机器人数据**，只是给机器人数据更高的采样权重。

为什么要这样做？实验证明：

- 纯机器人数据微调 → 灾难性遗忘，所有语义泛化能力消失
- 纯 VLM 数据 → 不会输出动作
- 联合微调 → **既保持了 VLM 的语义泛化，又学会了输出动作**

这与 [[PaLM-E]] 中"保留语言-only 数据防止遗忘"的发现一脉相承。

### 2.2 Output Constraint（输出约束）

训练时模型可以输出任何 token（语言或动作）。但**推理时**，当模型被问到"机器人应该做什么动作"时，作者约束输出词汇表**只包括 256 个有效动作 token**——确保模型不会输出一段废话来替代动作。

### 2.3 实时推理的工程挑战

RT-2-55B 是当时用于直接闭环控制的最大模型（比此前最大的约大一个数量级）。无法在机器人本地运行。解决方案：**部署在多 TPU 云服务上，通过网络查询。**

- 55B 版本：1-3 Hz
- 5B 版本：~5 Hz

虽然对很多任务来说 1-3Hz 已经不够快，但这是 VLA 实时控制的第一次可行性验证。

## 三、实验设计的精妙之处

RT-2 的实验设计包含 6,000 次真实世界评估，覆盖四类泛化场景：

### 3.1 已见任务 (Seen Tasks)

200+ 个训练数据中已出现的指令，但物体位置、时间、光照都会变化。RT-2 和 RT-1 表现接近——因为这是"记忆"而非"泛化"的测试。

### 3.2 泛化到未见

**未见物体 (Unseen Objects)**：训练时没见过的新物体（如玩具、独特形状的瓶子）
- RT-2-PaLI-X: **62%** vs RT-1: 29%

**未见背景 (Unseen Backgrounds)**：新的桌面、新的壁纸
- RT-2-PaLM-E: **60%** vs RT-1: 32%

**未见环境 (Unseen Environments)**：完全不同的房间——办公室 vs 厨房
- RT-2-PaLI-X: **52%** vs RT-1: 26%

平均而言，RT-2 在泛化任务上比 RT-1 **翻倍**，比只用预训练视觉表征的 VC-1 和 R3M **高 6 倍**。

### 3.3 涌现能力 (Emergent Capabilities)——最令人兴奋的部分

RT-2 展现出训练数据中**不存在**的能力：

1. **符号理解**：把一个物体放到写有特定数字或图标的盘子上——机器人训练数据中没有过"数字/图标"标签
2. **关系推理**："拿起最小的物体"、"拿起离蓝色杯子最近的东西"——训练数据中从来没有"最小"、"最近"这类指令
3. **人识别**："把可乐递给 Taylor Swift"——训练数据中没有特定人物
4. **Chain-of-Thought 推理**：在命令中加入 CoT 提示（"想想步骤"），RT-2 能做多步推理
   - "我累了，什么饮料最好？" → RT-2 能推理出"能量饮料"然后抓起红牛
   - "什么可以用作临时锤子？" → RT-2 能推理出"石头"然后抓起石头

**这些能力从哪来？** 显然不是从机器人训练数据——机器人数据里没有"最小"、"最近"、"能量饮料"、"Taylor Swift"。它们来自 VLM 在互联网数据上学到的语义知识。RT-2 的创新就在于**让这些语义知识直接流入到动作生成中**。

### 3.4 规模的影响

RT-2 比较了 5B 和 55B 两个版本：

- 已见任务：两者接近
- 泛化到未见：55B **显著更好**
- 涌现技能：55B >> 5B（尤其是在复杂的语义推理任务上）

> **规模不仅让模型"更好"，还让模型"能做一些小模型根本不会的事情"。** 这是"涌现"的典型特征。

### 3.5 开放源码基线：Language Table

为了让开源社区可以复现和比较，RT-2 还在 Language Table 仿真环境中做了实验。用一个小 PaLI-3B 模型做联合微调，在 Language Table 上实现了显著超越已有方法的性能。

## 四、方法的局限

RT-2 虽然开创了一个领域，但有几个不可忽视的局限：

1. **完全闭源**：代码、权重、训练数据都不公开。后续社区花了近一年才出现 OpenVLA 这样的开源替代
2. **推理速度慢**：55B 版本 1-3Hz 对很多实时任务不够
3. **需要云 TPU 推理**：无法在机器人本地运行，增加了网络延迟和可靠性问题
4. **仅支持单臂 7DoF**：不支持双手操作、移动底盘、灵巧手
5. **动作 token 化粗糙**：256-bin 均匀离散化，对高精度任务可能不够
6. **训练数据有限**：相比后来 OXE 的 1M+ 轨迹，RT-2 只用了一个办公室厨房的数据（尽管是 17 个月收集的）

---

## 五、对后续工作的影响

RT-2 开创的范式被后续几乎所有 VLA 继承和改进：

| RT-2 的思想 | 被谁继承和改进 |
|------------|-------------|
| "动作即文本 token" | [[OpenVLA]]（覆写 Llama tokenizer 的 256 个最不常用 token）|
| "VLM 骨干 + 动作微调" | [[π0]]（PaliGemma + Flow Matching 动作专家）|
| Co-Fine-Tuning | [[OpenVLA]] 参考了但做了简化（纯机器人数据微调，跳过了 VLM 数据混合）|
| 涌现能力 | [[MolmoAct]] 将其结构化（深度感知 → 轨迹规划 → 动作，使推理过程可解释）|

---

## 六、对你的启示

1. **"动作即语言"是目前最成功的 VLA 范式**——虽然 Diffusion Policy/Flow Matching 在动作生成上更优，但"动作 token 化 + LLM 骨干"依然是语义泛化的最强组合
2. **语义泛化来自互联网预训练**——没有它，RT-2 就跟 RT-1 没什么区别
3. **规模真的很重要**：55B > 5B，尤其在"你没想到的能力"上
4. 但你不需要重复 55B——[[OpenVLA]] 证明了 7B（甚至 BitVLA 的 3B）可以接近甚至超越 55B 的性能
