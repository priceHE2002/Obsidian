## 第 1 章 计算机系统概述

### 1.1 计算机系统的基本组成

计算机系统由硬件系统和软件系统共同组成。硬件提供数据存储、运算、控制和输入输出能力；软件通过指令序列组织硬件完成任务。

冯·诺依曼结构的核心思想：

- 程序和数据以二进制形式存放在主存中。
- 指令按地址顺序存放，通常顺序执行，遇到转移、调用、中断时改变执行流。
- CPU 由运算器和控制器组成，通过总线与主存、I/O 设备交换信息。
- 指令执行过程本质上是不断进行“取指、分析、执行”的循环。

![计算机工作过程](</Users/heyuhang/Desktop/计算机组成原理截图/计算机工作过程.png>)

### 1.2 CPU 的基本功能

CPU 的主要功能可以归纳为五类：

- 指令控制：按程序规定顺序取指、分析指令、执行指令。
- 操作控制：产生控制信号，使寄存器、ALU、主存、I/O 等部件协同动作。
- 时间控制：按照时钟和节拍安排各微操作的先后顺序。
- 数据加工：完成算术运算和逻辑运算。
- 中断处理：响应异常情况或外部请求，保存断点并转入中断服务程序。

![CPU的功能](</Users/heyuhang/Desktop/计算机组成原理截图/CPU的功能.png>)

### 1.3 程序执行的宏观流程

一条“取数”类指令的典型执行过程：

1. 根据 PC 给出的地址从主存取出指令。
2. 指令送入 IR。
3. CU 对操作码 OP 译码。
4. 根据地址码 Ad 访问操作数。
5. 将操作数送入目标寄存器或参与 ALU 运算。
6. PC 更新到下一条指令，或由转移类指令改写。

常见寄存器含义：

- `PC`：程序计数器，存放下一条将要执行指令的地址。
- `IR`：指令寄存器，存放当前正在执行的指令。
- `MAR`：主存地址寄存器，存放将要访问的主存地址。
- `MDR`：主存数据寄存器，暂存从主存读出或写入主存的数据。
- `ACC`：累加器，常用于保存操作数或运算结果。

## 第 2 章 数据的表示和运算

### 2.1 逻辑门电路基础

数字电路中的基本逻辑门包括与、或、非、与非、或非、异或、同或。考查时通常需要掌握逻辑表达式、真值表和门电路图形符号之间的对应关系。

![逻辑门电路基础总结](</Users/heyuhang/Desktop/计算机组成原理截图/逻辑门电路基础总结.png>)

速记：

- 与门：全 1 才为 1。
- 或门：有 1 就为 1。
- 非门：0 变 1，1 变 0。
- 异或：输入不同为 1。
- 同或：输入相同为 1。

### 2.2 原码、反码、补码、移码

定点整数常用编码：

- 原码：最高位为符号位，数值位表示绝对值。
- 反码：正数与原码相同；负数符号位不变，数值位按位取反。
- 补码：正数与原码相同；负数在反码基础上末位加 1。
- 移码：常用于阶码表示，只能表示整数；通常由补码符号位取反得到。

![原反补移码转换关系](</Users/heyuhang/Desktop/计算机组成原理截图/原反补移码转换关系.png>)

重点：

- 计算机中带符号整数通常用补码表示。
- 补码的好处是加减法可以统一为加法。
- n 位补码范围：`-2^(n-1) ~ 2^(n-1)-1`。
- n 位无符号整数范围：`0 ~ 2^n-1`。

### 2.3 无符号数加减运算

无符号数加法从最低位开始按位相加，向高位进位。最高位产生进位时，表示结果超过 n 位无符号数能表示的最大值。

无符号数减法通常转化为加法：

```text
A - B = A + (-B)
```

在二进制补码硬件中，可对减数逐位取反并加 1。

![无符号数加减运算](</Users/heyuhang/Desktop/计算机组成原理截图/无符号数加减运算.png>)

溢出判断：

- 无符号加法：最高位进位 `Cout = 1`，表示溢出。
- 无符号减法：减法转加法后，若最高位进位 `Cout = 0`，表示不够减，即发生溢出。
- 手算判断：结果超出 `0 ~ 2^n-1` 就溢出。

### 2.4 加减运算电路与标志位

ALU 每次运算后会产生状态标志，通常保存在 PSW 或 FLAGS 中。

![加减运算电路](</Users/heyuhang/Desktop/计算机组成原理截图/加减运算电路.png>)

常见标志位：

- `OF`：溢出标志，用于判断带符号数加减是否溢出。
- `SF`：符号标志，结果最高有效位为 1 时通常表示负数。
- `ZF`：零标志，结果为 0 时置 1。
- `CF`：进位/借位标志，常用于判断无符号数运算是否溢出。

带符号加减溢出判断：

- 两个正数相加得到负数：溢出。
- 两个负数相加得到正数：溢出。
- 正负相加不会发生带符号溢出。

### 2.5 定点数移位运算

移位分为逻辑移位和算术移位：

- 逻辑左移：高位移出丢弃，低位补 0。
- 逻辑右移：低位移出丢弃，高位补 0。
- 算术左移：规则与逻辑左移相同，但要注意符号位变化可能导致溢出。
- 算术右移：低位移出丢弃，高位补符号位。

![定点数的移位运算](</Users/heyuhang/Desktop/计算机组成原理截图/定点数的移位运算.png>)

与乘除的关系：

- 左移 r 位：相当于乘以 `2^r`。
- 右移 r 位：相当于除以 `2^r`。
- 对带符号数，算术右移才保持符号意义。

### 2.6 数的比较与 cmp 指令

硬件通常不直接“比较大小”，而是做减法并根据标志位判断。

```asm
cmp a, b
```

本质上执行 `a - b`，但不保存减法结果，只更新标志位。

![硬件如何实现数的比较](</Users/heyuhang/Desktop/计算机组成原理截图/硬件如何实现数的比较.png>)

![cmp指令的底层原理](</Users/heyuhang/Desktop/计算机组成原理截图/cmp指令的底层原理.png>)

常见条件转移判断：

- `je`：相等跳转，通常看 `ZF = 1`。
- `jne`：不等跳转，通常看 `ZF = 0`。
- `jg`：带符号大于，通常看 `ZF = 0 且 SF = OF`。
- `jge`：带符号大于等于，通常看 `SF = OF`。
- `jl`：带符号小于，通常看 `SF != OF`。
- `jle`：带符号小于等于，通常看 `ZF = 1 或 SF != OF`。

> [!warning] 易错点
> 无条件转移 `jmp` 不依赖 PSW 标志位；条件转移才需要根据标志位判断是否改变 PC。

### 2.7 乘法运算

计算机实现乘法常见三种方式：

- 阵列乘法器：速度最快，可在一个时钟周期内完成，但硬件复杂。
- ALU、移位器、寄存器、控制逻辑组合：速度较快，通常需要多个时钟周期。
- 用移位和加减法模拟：硬件简单，速度最慢。

![乘法的三种方式的对比](</Users/heyuhang/Desktop/计算机组成原理截图/乘法的三种方式的对比.png>)

#### 2.7.1 无符号整数乘法

无符号乘法的基本思想：

1. 判断乘数最低位。
2. 若最低位为 1，则将被乘数加到部分积中。
3. 若最低位为 0，则不加。
4. 将部分积和乘数整体右移。
5. 重复 n 轮。

![无符号整数乘法原理](</Users/heyuhang/Desktop/计算机组成原理截图/无符号整数乘法原理.png>)

硬件结构通常包括：

- 被乘数寄存器 `X`。
- 乘积高位寄存器 `P`。
- 乘数寄存器 `Y`。
- 进位触发器 `C`。
- ALU。
- 控制逻辑和计数器 `Cn`。

![32位无符号数乘法运算电路结构](</Users/heyuhang/Desktop/计算机组成原理截图/32位无符号数乘法运算电路结构.png>)

#### 2.7.2 带符号整数乘法

带符号整数常用补码乘法。典型思想是根据乘数最低位和辅助位的组合决定本轮操作：

- `辅助位 - Y最低位 = 1`：加 `[X]补`。
- `辅助位 - Y最低位 = 0`：加 0。
- `辅助位 - Y最低位 = -1`：减 `[X]补`。

随后将乘积寄存器、乘数寄存器和辅助位整体算术右移。

![带符号整数乘法原理](</Users/heyuhang/Desktop/计算机组成原理截图/带符号整数乘法原理.png>)

### 2.8 除法运算

无符号除法通常支持 `2n bit ÷ n bit`，最终得到 `n bit` 商和 `n bit` 余数。

![无符号整数除法原理](</Users/heyuhang/Desktop/计算机组成原理截图/无符号整数除法原理.png>)

基本步骤：

1. 先左移，空出的低位用于上商。
2. 根据中间余数与除数的比较决定本轮上商 1 还是 0。
3. 根据结果可能执行加法或减法修正余数。
4. 计数器减 1，重复直到 n 轮结束。

#### 2.8.1 商溢出

双精度除法 `2n bit ÷ n bit` 的商可能超过 n 位，因此可能发生商溢出。单精度无符号除法 `n bit ÷ n bit` 的商不会超过 n 位，通常不发生商溢出。

![商溢出的进一步探讨](</Users/heyuhang/Desktop/计算机组成原理截图/商溢出的进一步探讨.png>)

#### 2.8.2 带符号补码除法

补码除法需要注意：

- 被除数为 n 位时，通常先符号扩展为 2n 位。
- 除数为 n 位，放入除数寄存器。
- 特殊情况：除数为 0 产生异常。
- 补码单精度除法中，只有“最小负数 ÷ -1”可能发生商溢出。

![带符号整数补码除法](</Users/heyuhang/Desktop/计算机组成原理截图/带符号整数（补码除法.png>)

### 2.9 IEEE 754 浮点数

IEEE 754 浮点数由三部分组成：

```text
数值 = (-1)^S × M × 2^E
```

- `S`：符号位。
- `E`：阶码真值。
- `M`：尾数。

#### 2.9.1 float 单精度

float 共 32 位：

- 符号位：1 位。
- 阶码：8 位，偏置值 127。
- 尾数：23 位。

规格化尾数默认隐藏小数点前的 `1`，即实际尾数为 `1.f`。

![float单精度浮点型的存储](</Users/heyuhang/Desktop/计算机组成原理截图/float单精度浮点型的存储.png>)

#### 2.9.2 非规格化浮点数

当阶码全 0 且尾数非 0 时，表示非规格化数：

- float：`± 2^-126 × 0.f`
- double：`± 2^-1022 × 0.f`

非规格化数用于表示更接近 0 的数，填补 0 和最小规格化数之间的空隙。

![非规格化浮点数的表示](</Users/heyuhang/Desktop/计算机组成原理截图/非规格化浮点数的表示.png>)

#### 2.9.3 表示范围与特殊值

特殊值：

- 阶码全 0，尾数全 0：`+0` 或 `-0`。
- 阶码全 0，尾数非 0：非规格化数。
- 阶码全 1，尾数全 0：`+∞` 或 `-∞`。
- 阶码全 1，尾数非 0：`NaN`。

![IEEE 754浮点数的表示范围](</Users/heyuhang/Desktop/计算机组成原理截图/IEEE 754浮点数的表示范围.png>)

double 共 64 位：

- 符号位：1 位。
- 阶码：11 位，偏置值 1023。
- 尾数：52 位。

![double表示范围](</Users/heyuhang/Desktop/计算机组成原理截图/double表示范围.png>)

#### 2.9.4 舍入模式

IEEE 754 常见四种舍入模式：

- 就近舍入：默认模式，类似“四舍六入五留双”。
- 正向舍入：向 `+∞` 舍入。
- 负向舍入：向 `-∞` 舍入。
- 截断法：向 0 舍入，直接舍弃多余尾数。

![IEEE 754的4种舍入模式](</Users/heyuhang/Desktop/计算机组成原理截图/IEEE 754的4种舍入模式.png>)

> [!tip] 考研常用判断
> 浮点数上溢通常置为无穷大并设置溢出标志；下溢若落入非规格化区间则用非规格化数表示，若太小接近 0 则按机器零处理。

## 第 3 章 存储系统

### 3.1 存储器芯片的基本原理

存储芯片通过地址线选择存储单元，通过数据线读写数据，通过片选和读写控制信号决定是否工作。

![存储器芯片的基本原理](</Users/heyuhang/Desktop/计算机组成原理截图/存储器芯片的基本原理.png>)

重要概念：

- 地址线 n 位：可寻址 `2^n` 个存储单元。
- 数据线宽度：等于存储字长。
- 总容量：`存储单元个数 × 存储字长`。
- `CS/CE`：片选或芯片使能信号。
- `WE`：写使能信号。

### 3.2 译码器与片选

CPU 送出地址信号后，可通过译码器产生片选信号。译码器的使能端可以控制片选信号的生效时间，避免地址尚未稳定时错误选中存储芯片。

![译码器补充](</Users/heyuhang/Desktop/计算机组成原理截图/译码器补充.png>)

### 3.3 存储器扩展

存储器扩展常见两类：

- 位扩展：增加存储字长。
- 字扩展：增加存储字数。

字扩展时，常用高位地址线产生片选信号。线选法用若干高位地址线直接作为片选信号，简单但地址空间利用率不高。

![字拓展-线选法](</Users/heyuhang/Desktop/计算机组成原理截图/字拓展-线选法.png>)

### 3.4 SRAM 与 DRAM

SRAM 使用双稳态触发器存储信息；DRAM 使用栅极电容存储电荷。

![栅极电容vs双稳态触发器](</Users/heyuhang/Desktop/计算机组成原理截图/栅极电容vs双稳态触发器.png>)

对比：

- SRAM：速度快、成本高、集成度低、功耗较高，常用于 Cache。
- DRAM：速度慢、成本低、集成度高、需要刷新，常用于主存。

![DRAM vs SRAM](</Users/heyuhang/Desktop/计算机组成原理截图/DRAM vs SRAM.png>)

### 3.5 DRAM 刷新

DRAM 电容会漏电，因此需要周期性刷新。刷新通常以行为单位进行，因为 DRAM 存储阵列采用行列地址组织。

![DRAM的刷新](</Users/heyuhang/Desktop/计算机组成原理截图/DRAM的刷新.png>)

常见刷新方式：

- 集中刷新：集中一段时间刷新全部行，期间不能正常访存，存在“死时间”。
- 分散刷新：每个存储周期都安排刷新，系统速度下降但没有集中死时间。
- 异步刷新：在刷新周期内均匀分散刷新请求，折中方案。

### 3.6 多模块存储器

多模块存储器通过多个存储体并行工作提高访存带宽。

![多体并行vs单体多字](</Users/heyuhang/Desktop/计算机组成原理截图/多体并行vs单体多字.png>)

多体并行存储器：

- 每个模块容量和速度相同。
- 各模块有独立的读写控制电路、地址寄存器和数据寄存器。
- 可以并行工作，也可以交叉工作。
- 常见低位交叉编址使连续地址分布到不同体中，有利于连续取指和取数。

单体多字存储器：

- 每个存储单元包含多个字。
- 一次读出多个连续字。
- 适合连续访问，但不能单独取其中某个字。

### 3.7 Cache 基本概念

Cache 利用程序访问的局部性原理：

- 时间局部性：最近访问过的数据，短时间内可能再次访问。
- 空间局部性：访问某地址后，其附近地址也可能被访问。

命中率 `H`：CPU 访问的信息已经在 Cache 中的比例。缺失率 `M = 1 - H`。

![Cache命中率分析](</Users/heyuhang/Desktop/计算机组成原理截图/Cache命中率分析.png>)

平均访问时间：

```text
先访问 Cache，未命中再访问主存：
t = Htc + (1 - H)(tc + tm)

同时访问 Cache 和主存，Cache 命中则停止主存访问：
t = Htc + (1 - H)tm
```

### 3.8 Cache 与主存映射

Cache 行中通常保存：

- 有效位。
- 标记。
- 整块数据。

![Cache-主存映射映射方式回顾](</Users/heyuhang/Desktop/计算机组成原理截图/Cache-主存映射映射方式回顾.png>)

#### 3.8.1 直接映射

主存块只能放入特定 Cache 行：

```text
Cache 行号 = 主存块号 mod Cache 总行数
```

地址结构：

```text
标记 + 行号 + 块内地址
```

![直接映射访存](</Users/heyuhang/Desktop/计算机组成原理截图/直接映射访存.png>)

优点：查找速度快，只比较一个标记。  
缺点：冲突多，Cache 空间利用不充分。

#### 3.8.2 全相联映射

主存块可放入 Cache 任意行。

地址结构：

```text
标记 + 块内地址
```

![全相联映射访存](</Users/heyuhang/Desktop/计算机组成原理截图/全相联映射访存.png>)

优点：Cache 空间利用充分，命中率高。  
缺点：查找标记最慢，需要比较所有行或使用相联比较硬件。

#### 3.8.3 组相联映射

主存块只能放入指定组，但在组内可放任意行：

```text
所属组号 = 主存块号 mod Cache 总组数
```

地址结构：

```text
标记 + 组号 + 块内地址
```

![组相联映射访存](</Users/heyuhang/Desktop/计算机组成原理截图/组相联映射访存.png>)

组相联是直接映射与全相联的折中。n 路组相联表示每组有 n 个 Cache 行。

### 3.9 多级 Cache

现代计算机通常采用多级 Cache：

- 越靠近 CPU，速度越快、容量越小。
- 越远离 CPU，速度越慢、容量越大。
- L1、L2、L3 逐级扩大容量并降低速度。

![多级Cache](</Users/heyuhang/Desktop/计算机组成原理截图/多级Cache.png>)

常见写策略：

- Cache 与 Cache 之间：常用全写法加非写分配法。
- Cache 与主存之间：常用写回法加写分配法。

### 3.10 外存与 SSD

SSD 基于 Flash Memory，属于 EEPROM。其存储介质由多个 Flash Chip 组成，每个芯片包含多个块，每个块包含多个页。

![SSD](</Users/heyuhang/Desktop/计算机组成原理截图/SSD.png>)

SSD 读写特征：

- 以页为单位读写，类似磁盘的扇区。
- 以块为单位擦除。
- 支持随机访问，读快写慢。
- 若要写入的页已有数据，需要先把块内有效页复制到新块，再擦除旧块。

磨损均衡：

- 动态磨损均衡：写入时优先选择擦除次数少的块。
- 静态磨损均衡：迁移冷数据，让较新的块承担更多写任务。

## 第 4 章 指令系统

### 4.1 指令格式

一条指令通常由操作码字段和地址码字段组成：

```text
操作码 OP + 地址码 A
```

操作码说明“做什么”，地址码说明“对谁做”或“结果放哪里”。

### 4.2 地址码个数

四地址指令格式：

```text
OP | A1 | A2 | A3 | A4
```

语义：

```text
(A1) OP (A2) -> A3
A4 = 下一条将要执行指令的地址
```

![四地址指令](</Users/heyuhang/Desktop/计算机组成原理截图/四地址指令.png>)

访存次数：

- 取指 1 次。
- 读 A1 1 次。
- 读 A2 1 次。
- 写 A3 1 次。
- 共 4 次访存，不含取 A4 所指指令的后续访存。

正常顺序执行时，PC 取指后自动加 1；四地址指令执行后会把 PC 改为 A4，显式指定下一条指令地址。

### 4.3 扩展操作码

扩展操作码用于在定长指令字中支持不同地址数的指令。设计时必须满足：

- 短操作码不能是长操作码的前缀。
- 各指令操作码不能重复。

![扩展操作码](</Users/heyuhang/Desktop/计算机组成原理截图/扩展操作码.png>)

常用策略：

- 使用频率高的指令分配短操作码。
- 使用频率低的指令分配长操作码。
- 这样可以减少平均译码和分析时间。

### 4.4 指令寻址

指令寻址回答“下一条要执行的指令在哪里”。

- 顺序寻址：由 PC 自动给出下一条指令地址。
- 跳跃寻址：由转移指令、调用指令、中断等改写 PC。

变长指令系统中，CPU 需要先读入一个字，根据操作码判断整条指令长度 n，再执行：

```text
(PC) + n -> PC
```

![变长指令顺序寻址](</Users/heyuhang/Desktop/计算机组成原理截图/变长指令顺序寻址.png>)

### 4.5 数据寻址

数据寻址回答“操作数在哪里”。指令中通常包含：

```text
操作码 OP + 寻址特征 + 形式地址 A
```

![数据寻址特征](</Users/heyuhang/Desktop/计算机组成原理截图/数据寻址特征.png>)

常见寻址方式：

- 隐含寻址：操作数隐含在指令约定的寄存器中。
- 立即寻址：形式地址字段本身就是操作数。
- 直接寻址：形式地址字段就是有效地址。
- 间接寻址：形式地址字段指向的主存单元中存放有效地址。
- 寄存器寻址：操作数在寄存器中。
- 寄存器间接寻址：寄存器中存放有效地址。
- 相对寻址：有效地址为 PC 与偏移量之和。
- 基址寻址：有效地址为基址寄存器与形式地址之和。
- 变址寻址：有效地址为变址寄存器与形式地址之和。
- 堆栈寻址：隐含使用栈指针 SP。

### 4.6 变址寻址

变址寻址适合数组和循环访问。通常把数组首地址作为形式地址或基址，变址寄存器存放下标或偏移量。

![变址寻址的作用](</Users/heyuhang/Desktop/计算机组成原理截图/变址寻址的作用.png>)

典型用途：

```c
for (int i = 0; i < 10; i++) {
    sum += a[i];
}
```

汇编层面可不断修改变址寄存器的值，形成 `数组首地址 + 下标偏移` 的有效地址。

### 4.7 堆栈寻址

堆栈按后进先出原则管理，隐含使用栈指针 `SP` 作为操作数地址。

![堆栈寻址](</Users/heyuhang/Desktop/计算机组成原理截图/堆栈寻址.png>)

若栈顶在低地址方向：

- 入栈：先 `SP - 1 -> SP`，再把数据写入 `M(SP)`。
- 出栈：先读出 `M(SP)`，再 `SP + 1 -> SP`。

若栈顶在高地址方向：

- 入栈：先 `SP + 1 -> SP`，再写入。
- 出栈：先读出，再 `SP - 1 -> SP`。

### 4.8 常见机器级指令

常见算术运算指令：

![常见的算术运算指令](</Users/heyuhang/Desktop/计算机组成原理截图/常见的算术运算指令.png>)

Intel 风格中常见写法：

- `add d, s`：`d + s -> d`
- `sub d, s`：`d - s -> d`
- `mul d, s`：无符号乘。
- `imul d, s`：有符号乘。
- `div s`：无符号除，常用 `edx:eax / s`。
- `idiv s`：有符号除。
- `neg d`：取负。
- `inc d`：自增。
- `dec d`：自减。

#### 4.8.1 mov 指令示例

#### 4.8.2 无条件转移 jmp

无条件转移直接改写 PC/IP，不需要查看标志位。

![无条件转移指令（带标号）](</Users/heyuhang/Desktop/计算机组成原理截图/无条件转移指令（带标号）.png>)

常见形式：


标号本质上用于锚定指令地址。

#### 4.8.3 条件转移实现循环

用条件转移实现循环通常包含四部分：

1. 循环前初始化。
2. 判断是否直接跳过循环。
3. 循环主体。
4. 判断是否继续循环。

![用条件转移指令实现循环](<./计算机组成原理截图/用条件转移指令实现循环.png>)

#### 4.8.4 loop 指令实现循环

`loop label` 常隐含使用 `ecx` 作为循环计数器：

```asm
loop Looptop
```

含义：

```text
ecx--
if ecx != 0: jump Looptop
```

![用loop指令实现循环](<./计算机组成原理截图/用loop指令实现循环.png>)

补充：

- `loopnz`：当 `ecx != 0` 且 `ZF = 0` 时继续循环。
- `loopz`：当 `ecx != 0` 且 `ZF = 1` 时继续循环。

### 4.9 AT&T 格式与 Intel 格式

![AT&T格式](<./计算机组成原理截图/AT&T格式.png>)

主要差别：

- 操作数顺序：AT&T 为 `op s, d`；Intel 为 `op d, s`。
- 寄存器：AT&T 寄存器前加 `%`，Intel 直接写寄存器名。
- 立即数：AT&T 立即数前加 `$`，Intel 直接写数字。
- 主存地址：AT&T 常用小括号，Intel 常用中括号。
- 读写长度：AT&T 常在指令后加 `b/w/l`，Intel 常用 `byte ptr/word ptr/dword ptr`。

## 第 5 章 中央处理器

### 5.1 CPU 的基本结构

CPU 通常由运算器、控制器、寄存器组和内部数据通路组成。

![CPU的基本结构](</Users/heyuhang/Desktop/计算机组成原理截图/CPU的基本结构.png>)

关键部件：

- 运算器：完成算术和逻辑运算。
- 控制器：分析指令并产生控制信号。
- 寄存器组：暂存操作数、地址、指令和中间结果。
- 数据通路：连接各部件并传递数据。

### 5.2 运算器

运算器用于实现算术运算和逻辑运算。常见组成：

- `ALU`：算术逻辑单元。
- `ACC`：累加器，存放操作数或运算结果。
- `MQ`：乘商寄存器，乘除运算中保存乘数、乘积低位或商。
- `X`：通用操作数寄存器。
- 移位器：对结果进行移位操作。
- `PSW`：保存运算状态标志。

![运算器的基本组成](</Users/heyuhang/Desktop/计算机组成原理截图/运算器的基本组成.png>)

### 5.3 CPU 内部总线方式

CPU 内部单总线方式把多个寄存器输入输出端连接到一条公共通路上。

优点：结构简单，容易实现。  
缺点：同一时刻总线上只能有一个部件输出，容易出现数据传输冲突，性能较低。

![计算器的基本结构-CPU内部总线](</Users/heyuhang/Desktop/计算机组成原理截图/计算器的基本结构-CPU内部总线.png>)

### 5.4 专用数据通路方式

专用数据通路根据指令执行过程中的数据流动方向安排连接线路。

![运算器的基本结构-专用数据通路方式](</Users/heyuhang/Desktop/计算机组成原理截图/运算器的基本结构-专用数据通路方式.png>)

优点：性能高，冲突少。  
缺点：硬件复杂，连线多，不易实现。

### 5.5 数据通路中的微操作

在 CPU 内部单总线方式中，微操作需要通过控制信号完成。

![数据通路-CPU内部单总线方式](</Users/heyuhang/Desktop/计算机组成原理截图/数据通路-CPU内部单总线方式.png>)

例：把 PC 内容送入 MAR：

```text
(PC) -> Bus      PCout 有效
Bus -> MAR       MARin 有效
```

例：从主存取指：

```text
(PC) -> Bus -> MAR
1 -> R
MEM(MAR) -> MDR
MDR -> Bus -> IR
```

例：执行加法指令：

```text
Ad(IR) -> Bus -> MAR
1 -> R
MEM(MAR) -> MDR
MDR -> Bus -> Y
(ACC) + (Y) -> Z
Z -> ACC
```

### 5.6 控制器基本组成

控制器的基本任务是分析指令、产生控制信号、协调各部件按时序完成微操作。

![控制器](</Users/heyuhang/Desktop/计算机组成原理截图/控制器.png>)

![控制器的基本结构](</Users/heyuhang/Desktop/计算机组成原理截图/控制器的基本结构.png>)

常见部件：

- `PC`：指出下一条指令地址。
- `IR`：保存当前指令。
- 指令译码器 `ID`：对操作码字段译码。
- 微操作信号发生器：根据 IR、PSW 和时序信号产生控制信号。
- 时序系统：产生节拍信号。
- `MAR/MDR`：负责 CPU 与主存之间的地址和数据交换。

### 5.7 指令周期

指令周期是 CPU 从主存中取出并执行一条指令所需的全部时间。

![指令周期](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期.png>)

关系：

- 指令周期包含若干机器周期。
- 机器周期又包含若干时钟周期。
- 时钟周期也称节拍、T 周期或 CPU 时钟周期，是 CPU 操作的基本时间单位。

不同指令的指令周期可不同：

![指令周期2](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期2.png>)

### 5.8 指令周期流程

典型流程：

1. 取指周期。
2. 判断是否有间址。
3. 若有间址，进入间址周期。
4. 执行周期。
5. 判断是否有中断。
6. 若有中断，进入中断周期。
7. 返回取下一条指令。

![指令周期流程](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期流程.png>)

四个工作周期的标志触发器：

- `FE`：取指周期。
- `IND`：间址周期。
- `EX`：执行周期。
- `INT`：中断周期。

### 5.9 取指周期数据流

取指周期的目的：从主存取出当前指令并送入 IR，同时形成下一条指令地址。

![指令周期的数据流-取指周期](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期的数据流-取指周期.png>)

微操作序列：

```text
(PC) -> MAR
1 -> R
M(MAR) -> MDR
(MDR) -> IR
(PC) + 1 -> PC
OP(IR) -> ID
```

### 5.10 间址周期数据流

间址周期的目的：取有效地址。

![指令周期的数据流-间址周期](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期的数据流-间址周期.png>)

微操作序列：

```text
Ad(IR) -> MAR
1 -> R
M(MAR) -> MDR
(MDR) -> Ad(IR)
```

若是多级间接寻址，可能重复间址周期。

### 5.11 中断周期数据流

中断周期的目的：保存断点并转入中断服务程序。

![指令周期的数据流-中断周期](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期的数据流-中断周期.png>)

典型微操作：

```text
(SP) - 1 -> SP
(SP) -> MAR
1 -> W
(PC) -> MDR
MDR -> M(MAR)
向量地址 -> PC
```

注意：保存断点通常保存的是 PC 内容，即下一条将要执行指令的地址。

### 5.12 微操作时序安排

安排微操作时序的原则：

- 先后顺序不能随意改变。
- 被控对象不同的微操作，尽量安排在同一节拍内完成。
- 占用时间短的微操作，尽量安排在同一节拍内完成。
- 占用时间长的微操作需要单独保证完成时间。

![安排微操作时序-取指周期](</Users/heyuhang/Desktop/计算机组成原理截图/安排微操作时序-取指周期.png>)

取指周期可安排为：

```text
T0: (PC) -> MAR, 1 -> R
T1: M(MAR) -> MDR, (PC) + 1 -> PC
T2: (MDR) -> IR, OP(IR) -> ID
```

### 5.13 指令执行方案

常见方案：

- 单指令周期：所有指令使用相同执行时间，周期取决于最慢指令，简单但效率低。
- 多指令周期：不同指令使用不同执行步骤和周期，效率更高但控制复杂。
- 流水线方案：多个指令并行处于不同执行阶段，提高吞吐率。

![指令执行方案](</Users/heyuhang/Desktop/计算机组成原理截图/指令执行方案.png>)

### 5.14 硬布线控制器

硬布线控制器用组合逻辑电路产生控制信号。

![硬布线控制器](</Users/heyuhang/Desktop/计算机组成原理截图/硬布线控制器.png>)

控制信号通常由以下因素共同决定：

- 指令操作码。
- 当前机器周期。
- 当前节拍信号。
- 机器状态条件。
- 来自执行部件、I/O 或主存的反馈标志。

特点：

- 速度快。
- 硬件复杂。
- 修改和扩展困难。

### 5.15 微程序控制器

微程序控制器把控制信号组织为微指令，把一条机器指令的执行过程组织为一段微程序。

![微程序控制器的设计思路](</Users/heyuhang/Desktop/计算机组成原理截图/微程序控制器的设计思路.png>)

核心概念：

- 微命令：控制一个或一组微操作的控制信号。
- 微操作：由微命令控制完成的基本操作。
- 微指令：若干微命令和顺序控制信息的集合。
- 微程序：若干微指令的序列，对应一条机器指令的执行过程。
- 控制存储器 `CM`：存放微程序。
- `CMAR`：控制存储器地址寄存器。
- `CMDR`：控制存储器数据寄存器，存放读出的微指令。

![微程序控制器的工作原理](</Users/heyuhang/Desktop/计算机组成原理截图/微程序控制器的工作原理.png>)

### 5.16 微指令格式

微指令通常包括：

```text
操作控制字段 + 顺序控制字段
```

![微指令的格式](</Users/heyuhang/Desktop/计算机组成原理截图/微指令的格式.png>)

水平型微指令：

- 一条微指令可定义多个可并行的微命令。
- 优点：微程序短，执行速度快。
- 缺点：微指令长，编写微程序较麻烦。

垂直型微指令：

- 一条微指令通常只定义一个微命令。
- 优点：微指令短、规整，便于编写。
- 缺点：微程序长，执行速度慢，效率低。

### 5.17 微指令地址形成方式

后继微指令地址可以通过多种方式形成：

![微指令的地址形成方式](</Users/heyuhang/Desktop/计算机组成原理截图/微指令的地址形成方式.png>)

常见方式：

- 微指令下地址字段直接给出。
- 根据机器指令操作码形成。
- 增量计数器法：`CMAR + 1 -> CMAR`。
- 分支转移：根据判别条件选择转移地址。
- 通过测试网络形成。
- 硬件产生微程序入口地址。

## 第 6 章 总线

### 6.1 总线分类

总线是多个功能部件之间传输信息的公共通路。按传输信息类型可分为：

- 地址总线：传输地址信息，通常由 CPU 发出。
- 数据总线：传输数据，通常双向。
- 控制总线：传输读写、中断、响应、时钟等控制信号。

在 CPU 与主存的数据流图中，地址总线、数据总线、控制总线分别承担不同角色：

![CPU的基本结构](</Users/heyuhang/Desktop/计算机组成原理截图/CPU的基本结构.png>)

### 6.2 总线事务

一次主存读操作通常包括：

1. CPU 将地址送上地址总线。
2. CPU 通过控制总线发出读命令。
3. 主存根据地址取出数据。
4. 主存将数据送上数据总线。
5. CPU 通过 MDR 接收数据。

一次主存写操作通常包括：

1. CPU 将地址送上地址总线。
2. CPU 将写入数据送上数据总线。
3. CPU 通过控制总线发出写命令。
4. 主存把数据写入对应存储单元。

### 6.3 同步与异步

同步总线使用统一时钟控制各阶段，时序简单，但受最慢部件限制。异步总线不使用统一时钟，常通过请求和回答信号握手，适合速度差异较大的设备，但控制更复杂。

### 6.4 总线带宽

总线带宽常表示单位时间内可传输的数据量：

```text
总线带宽 = 总线宽度 × 总线频率 × 每周期传输次数
```

解题时要注意单位换算：

- bit 与 Byte：`1B = 8bit`。
- MB 与 MiB 在考研题中通常按题目给定处理。
- 若是双倍速、四倍速传输，要乘以每周期传输次数。

## 第 7 章 输入/输出系统

### 7.1 I/O 系统基本概念

I/O 系统负责 CPU、主存和外部设备之间的数据交换。外部设备速度差异大，通常需要接口电路完成：

- 数据缓冲。
- 地址译码。
- 控制和状态管理。
- 中断请求。
- 数据格式转换。

### 7.2 程序查询方式

程序查询方式由 CPU 主动不断检查设备状态，设备准备好后再进行数据交换。

优点：硬件简单。  
缺点：CPU 等待时间长，效率低。

### 7.3 中断方式

中断方式下，设备准备好后向 CPU 发出中断请求。CPU 在当前指令执行完成后响应中断，保存断点并转入中断服务程序。

中断周期与 CPU 指令周期密切相关：

![指令周期流程](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期流程.png>)

![指令周期的数据流-中断周期](</Users/heyuhang/Desktop/计算机组成原理截图/指令周期的数据流-中断周期.png>)

中断处理核心：

- 保存断点：通常保存 PC。
- 保护现场：保存通用寄存器和 PSW 等。
- 执行中断服务程序。
- 恢复现场。
- 返回原程序继续执行。

### 7.4 DMA 方式

DMA 方式允许 I/O 设备在 DMA 控制器管理下直接与主存交换数据，减少 CPU 逐字传输负担。

DMA 基本过程：

1. CPU 初始化 DMA 控制器，给出主存地址、设备地址、传输方向和传输长度。
2. DMA 控制器申请总线控制权。
3. DMA 控制器直接完成主存与设备之间的数据传输。
4. 传输结束后向 CPU 发出中断。

与中断方式区别：

- 中断方式仍由 CPU 执行数据传送指令。
- DMA 方式由 DMA 控制器完成数据块传输，CPU 只参与初始化和收尾。

## 考前速查

### A. 指令周期微操作

取指周期：

```text
(PC) -> MAR
1 -> R
M(MAR) -> MDR
(MDR) -> IR
(PC) + 1 -> PC
OP(IR) -> ID
```

间址周期：

```text
Ad(IR) -> MAR
1 -> R
M(MAR) -> MDR
(MDR) -> Ad(IR)
```

中断周期：

```text
(SP) - 1 -> SP
(SP) -> MAR
1 -> W
(PC) -> MDR
MDR -> M(MAR)
向量地址 -> PC
```

### B. Cache 公式

平均访问时间：

```text
串行访问：t = Htc + (1 - H)(tc + tm)
并行访问：t = Htc + (1 - H)tm
```

映射规则：

```text
直接映射：Cache 行号 = 主存块号 mod Cache 总行数
组相联：所属组号 = 主存块号 mod Cache 总组数
全相联：主存块可放任意 Cache 行
```

地址结构：

```text
直接映射：标记 + 行号 + 块内地址
全相联：标记 + 块内地址
组相联：标记 + 组号 + 块内地址
```

### C. 标志位

- `OF`：带符号溢出。
- `SF`：结果符号。
- `ZF`：结果是否为 0。
- `CF`：无符号进位或借位。

判断：

- 无符号加法溢出：最高位进位为 1。
- 无符号减法溢出：减法转加法后最高位进位为 0。
- 带符号加法溢出：同号相加，结果异号。
- `cmp a, b`：本质做 `a - b`，只改标志位不保存结果。

### D. 栈与函数调用

常见栈操作：

```asm
push x        ; esp -= 4, x -> M[esp]
pop x         ; M[esp] -> x, esp += 4
```

函数入口：

```asm
push ebp
mov ebp, esp
sub esp, n
```

函数出口：

```asm
mov esp, ebp
pop ebp
ret
```

调用与返回：

- `call`：压入返回地址，跳转到被调用函数。
- `ret`：弹出返回地址，恢复 IP/PC。

栈帧访问：

- 局部变量：`[ebp-4]`、`[ebp-8]`。
- 参数：`[ebp+8]`、`[ebp+12]`。
- 返回值：x86 常用 `eax`。

### E. 浮点数

float：

```text
1 位符号 + 8 位阶码 + 23 位尾数
偏置值 = 127
```

double：

```text
1 位符号 + 11 位阶码 + 52 位尾数
偏置值 = 1023
```

特殊值：

```text
阶码全 0，尾数全 0：±0
阶码全 0，尾数非 0：非规格化数
阶码全 1，尾数全 0：±∞
阶码全 1，尾数非 0：NaN
```

### F. 高频易错点

- PC 中保存的是下一条将要执行指令的地址，不一定是当前指令地址。
- 取指结束后通常有 `(PC)+1 -> PC`，变长指令应根据指令长度更新 PC。
- 四地址指令中的 `A4` 是下一条指令地址，执行后会改写 PC。
- 间接寻址会多一次或多次访存来取有效地址。
- 堆栈增长方向不同，入栈和出栈时 SP 增减方向也不同。
- `CF` 主要看无符号数，`OF` 主要看带符号数。
- Cache 题先求块内地址位数，再求行号/组号位数，剩余才是标记位。
- 全相联没有行号字段；直接映射没有组内选择；组相联要分清“组数”和“路数”。
- DRAM 需要刷新，SRAM 不需要刷新。
- 微程序控制器中的“下地址字段”指出后继微指令地址，不是机器指令地址。
- `call` 压栈的是返回地址，`ret` 弹出的是返回地址。
- Intel 与 AT&T 操作数顺序相反。
